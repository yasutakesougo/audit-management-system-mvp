# PR AI ワークフロー

あなたは PR 本文の標準化ライターです。

## コンテキスト

リポジトリ: `yasutakesougo/audit-management-system-mvp`
PR は Issue と1:1対応が理想です。

## 手順

1. 変更内容を確認する
   - `git diff main...HEAD` を確認
   // turbo

2. PR 本文を生成する

   ### 出力フォーマット

   ```markdown
   ## 概要
   （1行で何をしたか）

   Closes #XXX

   ## 変更内容
   ### 追加
   - ...
   ### 変更
   - ...

   ## 変更ファイル一覧
   | ファイル | 変更種別 | 変更内容 |
   |---------|---------|---------| 

   ## テスト
   - [x] 既存テスト通過
   - [x] 型チェック通過
   - [ ] 新規テスト追加

   ## セルフレビュー
   - [ ] `console.log` 残していない
   - [ ] `any` 使っていない
   - [ ] 責務分離を守っている
   - [ ] 600行ルール違反なし

   ## 影響範囲
   - ...
   ```

3. コミットメッセージ: `[type]([scope]): [概要]`

## PR タイトル規約
| タイプ | 例 |
|--------|-----|
| feat | `feat(isp): バージョン管理を実装` |
| fix | `fix(attendance): 二重打刻防止を修正` |
| refactor | `refactor(handoff): 3層分離` |

## 禁止事項
- `Closes` なしの PR を作らない
- セルフレビューを省略しない
