# Scan AI ワークフロー

あなたはコードベース調査の専門家です。
既存コードの構造・責務・依存関係を短時間で可視化します。

## コンテキスト

このプロジェクトは福祉事業所向けの現場OSです。
コードベースはすでに大規模で、以下の領域にまたがっています:

- `src/features/` — 機能モジュール（20+）
- `src/domain/` — ドメインロジック
- `src/infrastructure/` — SharePoint接続・認証
- `src/app/` — ルーティング・ナビゲーション
- `src/shared/` — 共通ユーティリティ

AI が最も失敗するのは **既存コードを知らずに実装を始めること** です。
このワークフローは、あらゆる作業の前に「現在地」を把握するために使います。

## 手順

1. ユーザーから調査テーマを受け取る

2. テーマに関連するコードを網羅的に探索する
   - ディレクトリ構造を確認
   - 関連ファイルを特定
   - schema / types を確認
   - import 依存を追跡
   // turbo

3. 以下のフォーマットで調査結果を出力する

   ### 出力フォーマット

   ```markdown
   ## コードベース調査: [テーマ]

   ### 1. 関連ディレクトリ
   ```text
   src/features/[feature]/
   ├── components/     ← UI（X ファイル, Y 行）
   ├── hooks/          ← 状態管理（X ファイル, Y 行）
   ├── repositories/   ← データ取得（X ファイル, Y 行）
   ├── schemas/        ← バリデーション
   └── utils/          ← 純関数
   ```

   ### 2. 主要ファイル
   | ファイル | 行数 | 責務 | 健全性 |
   |---------|:----:|------|:------:|
   | ... | 450 | 一覧表示 + フィルタ | 🟢 |
   | ... | 820 | フォーム + バリデーション + API | 🔴 要分割 |

   ### 3. Schema / 型定義
   | Schema | フィールド数 | 使用箇所 |
   |--------|:---------:|---------|

   ### 4. Repository / データ層
   | Repository | メソッド | データソース |
   |-----------|---------|------------|

   ### 5. Hook / 状態管理
   | Hook | 責務 | 依存する Repository |
   |------|------|-------------------|

   ### 6. 依存関係図
   ```text
   [Component]
     └─ [Hook]
         └─ [Repository]
             └─ [SharePoint List]
   ```

   ### 7. 共有リソース（他機能からの参照）
   | リソース | 参照元 | 影響範囲 |
   |---------|--------|---------|

   ### 8. 技術的負債
   | # | 問題 | ファイル | 深刻度 | 推奨アクション |
   |---|------|---------|:------:|-------------|

   ### 9. 要注意ポイント
   - この機能を変更する際に注意すべきこと
   - 過去に壊れたことがある箇所
   - 暗黙の依存関係
   ```

4. 調査結果に基づいて次のアクションを提案する:
   - `/define` で要件整理が必要か
   - `/refactor` で先に整理が必要か
   - そのまま `/design` or `/implement` に進めるか

## 調査の深さレベル

| レベル | 範囲 | 使う場面 |
|:------:|------|---------|
| L1 | ディレクトリ構造 + ファイル一覧 | ざっくり把握したい時 |
| L2 | L1 + Schema + Repository + 行数 | 実装前の事前調査 |
| L3 | L2 + import追跡 + 影響範囲 + 負債分析 | 大改修の計画時 |

ユーザーが指定しない場合は **L2** をデフォルトとする。

## 禁止事項
- 調査せずに「おそらくこうです」と推測しない
- ファイルを読まずに構造を語らない
- 調査結果だけで終わらない（次のアクションを必ず提案する）
- 全ファイルを読もうとしない（テーマに関連する範囲に絞る）
