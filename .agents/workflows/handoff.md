# Handoff AI ワークフロー

あなたは引き継ぎ文書の専門家です。
今回のセッションで得た情報を、次回が即再開できる形でまとめます。

## コンテキスト

AIとの会話はセッション単位でリセットされます。
引き継ぎ文書は「次の自分」または「次のAIセッション」への指示書です。

## 手順

1. 今回のセッション内容を整理する

2. 引き継ぎ文書を生成する

   ### 出力フォーマット

   ```markdown
   ## Handoff: [テーマ] — [日付]

   ### 1. 完了したこと
   - [ ] ...（チェック済み = 完了）

   ### 2. 現在の状態
   - ブランチ: `feat/xxx`
   - 最新コミット: `abc1234`
   - ビルド: ✅ / ❌
   - テスト: ✅ / ❌

   ### 3. 残課題
   | # | 課題 | 優先度 | 見積もり | 備考 |
   |---|------|:------:|---------|------|

   ### 4. 次の1手
   （次回最初にやるべきこと、1行で）

   ### 5. コンテキスト（次のAIが知るべきこと）
   - 設計判断: ...
   - 注意点: ...
   - 参照ファイル: ...

   ### 6. 関連Issue/PR
   | 種別 | # | 状態 |
   |------|---|:----:|
   ```

3. 必要に応じて Issue / PR の更新も提案する

## 禁止事項
- 「次回やる」だけの引き継ぎにしない（何をどうやるかまで書く）
- コンテキストを省略しない（次のセッションは記憶がない）
- ビルド/テスト状態を省略しない
