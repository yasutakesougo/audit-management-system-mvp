# Add Regression Test Template

目的: 不具合修正や既存ガードに対して、再発防止テストを最小差分で追加する。

## 基本方針

- まず既存仕様、既存テスト、対象コードを読む。
- 実装変更よりテスト追加を優先する。
- 実装変更が必要な場合は、理由と影響範囲を明記する。
- テスト名は仕様を説明する名前にする。
- 外部サービス、SharePoint schema、環境設定、依存関係には触れない。
- 既存差分を巻き込まない。

## 作業手順

1. `git status --short` を確認する。
2. 失敗・再発防止したい仕様を1文で定義する。
3. 既存の近いテストファイルを探す。
4. 既存パターンに合わせてテストを1件または最小件数だけ追加する。
5. 対象テストを最小範囲で実行する。
6. 可能なら `npm run typecheck`、`npm run lint`、`git diff --check` を実行する。
7. 実装変更をした場合は、なぜテストだけでは足りなかったかを報告する。

## テスト名の指針

- `prevents ...`
- `rejects ...`
- `keeps ...`
- `falls back to ...`
- `does not ... when ...`

## 最終報告フォーマット

- 固定した仕様:
- 追加したテスト:
- 変更ファイル:
- 実装変更の有無:
- 実行した検証コマンド:
- PASS / FAIL:
- FAILがある場合の原因: 今回差分起因 / 既存問題 / 未確認
- リスク:
- PRタイトル案:
- PR本文案:
