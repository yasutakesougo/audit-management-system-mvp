# stabilization_7day_field_check_sheet.md

## 📘 目的

本チェックシートは、SharePoint連携システムの安定稼働を確実にするための
**7日間の観測・検証用チェックリスト**です。

技術的な正常だけでなく、**現場実態との一致（サイレント障害の検知）**を重視します。

---

# 🗓 運用ルール

* 毎日1回（推奨：朝）実施
* 所要時間：5〜10分
* 異常があれば即記録・共有（Slack / Teams）

---

# Day 1〜Day 7 共通チェック

---

## ① /admin/status（システム診断）

### ✅ チェック項目

* [ ] `missing_required` が 0件
* [ ] FAIL が発生していない
* [ ] WARN の件数が前日より増えていない
* [ ] 新しいエラー文言が出ていない

### 🔍 観点

* 「正常か」ではなく **「昨日と変わっていないか」**
* 小さな変化＝将来の障害予兆

---

## ② 現場突合（最重要）

### ✅ チェック項目

* [ ] 本日の利用者数と画面表示数が一致
* [ ] 記録件数と実際の記録が一致
* [ ] 「入力したのに表示されない」がない
* [ ] 「昨日のデータが消えた」がない

### 🔍 観点

* 技術ではなく **現場の事実が正しいか**
* ここで9割の事故を防げる

---

## ③ SUPPORT_RECORD_DETAILS 書き込み確認

### ✅ チェック項目

* [ ] 記録保存後に詳細行が生成されている
* [ ] 一部の行だけ欠けていない
* [ ] 表示と実データが一致している

### ⚠️ 注意（重要）

以下が起きたら即報告：

* 保存成功なのに一部データがない
* 詳細が欠落している

👉 **サイレント障害の典型パターン**

---

## ④ サイレント障害検知

### ✅ チェック項目

* [ ] 「遅い」「変」「消えた」などの声がない
* [ ] UIは正常だが違和感がない
* [ ] 再読み込みで直る現象が発生していない

### 🔍 観点

* エラーより「違和感」が重要
* Fail-Open設計の副作用を捕まえる

---

## ⑤ Nightly Patrol（前日ログ確認）

### ✅ チェック項目

* [ ] `sp:row_skipped` が増えていない
* [ ] `sp:fetch_fallback_success` が異常増加していない
* [ ] `detail_create_failed` が発生していない

### 🔍 観点

* 見えない異常をログで補足
* 「静かに壊れる」を防ぐ

---

* 日付：
* 事象：
* 発生画面：
* 再現性：あり / なし
* 影響範囲：
* 応急対応：
* スクリーンショット：

---

# 🚨 即時対応が必要な事象（P1）

以下は即対応：

* データ消失
* 保存成功なのに反映されない
* 利用者が表示されない
* 詳細データ欠落

---

# 🎯 7日後の判断基準

### ✅ 安定と判断できる状態

* 7日間連続で FAIL = 0
* WARN が増加していない
* 現場不整合ゼロ
* サイレント障害ゼロ
