# 🚀 本番投入最終判定 (Go/No-Go) テンプレート

本ドキュメントは、システムを本番稼働（または7日間の安定化フェーズ）に投入するかどうかを一発で判定するためのチェックリストです。

---

## 🚦 判定フェーズ

### 🟦 Phase 1: 静的検証 (Static Integrity)
コードベースが「動く状態」でビルドされているかを確認します。

| 項目 | 確認コマンド | 期待結果 | 判定 |
| :--- | :--- | :--- | :---: |
| **Type Check** | `npm run typecheck` | エラーなし (Exit 0) | [ ] |
| **Unit Test** | `npm run test` | 全テスト通過 | [ ] |
| **Build** | `npm run build` | 正常終了 (warnings 許容) | [ ] |

### 🟩 Phase 2: 実データ疎通 (Operational Flow)
実運用で最も発生しやすい「MultiChoice 関連の例外」を重点的に確認します。

| 項目 | 確認手順 | 期待結果 | 判定 |
| :--- | :--- | :--- | :---: |
| **新規保存** | `Staff_Master` 等で新規登録 | 保存が成功し、一覧に出る | [ ] |
| **Multi-空保存** | `WorkDays` 等を空にして更新 | 400エラーにならず空で保存される | [ ] |
| **再表示/結合** | 保存したデータを詳細画面で開く | 正しくマッピング・表示される | [ ] |

### 🟨 Phase 3: 診断ステータス (Diagnostics)
システム全体の Health 状態を確認します。

| 項目 | 確認場所 | 期待結果 | 判定 |
| :--- | :--- | :--- | :---: |
| **Essential Check** | AdminStatus (`/admin/status`) | `FAIL` が 0 件であること | [ ] |
| **Schema Drift** | AdminStatus (`/admin/status`) | `WARN` の内容を把握している | [ ] |
| **Audit Logs** | DebugConsole | `ERROR` の連発がないこと | [ ] |

---

## 🏁 最終判定マトリクス

| 判定結果 | 条件 | 推奨アクション |
| :--- | :--- | :--- |
| **🚀 GO** | 全項目が [x] かつ `FAIL` 0件 | **安定化フェーズ開始。** 監視を継続。 |
| **⚠️ CONDITIONAL READY** | Phase 1/2 は合格だが Phase 3 で一部 `WARN` あり | **本番稼働可能。** ただし管理者による週1回の drift 確認を条件とする。 |
| **🛑 NOT READY** | Phase 1/2 のいずれかが未達、または `FAIL` あり | **投入禁止。** `FAIL` 原因の解決（プロビジョニングまたは修正）が必要。 |
