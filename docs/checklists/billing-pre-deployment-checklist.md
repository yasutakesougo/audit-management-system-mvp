# Billing (請求・精算) 本番デプロイ前チェックリスト

## 📋 環境変数設定確認 (VITE_SP_LIST_BILLING_ORDERS)

本番想定での `/billing` ルート動作において、SharePoint の別サイト (`/sites/2`) にあるコーヒー注文履歴 (`List3`) を正しく参照するための環境設定です。

### 1. 本番環境変数設定値

以下の環境変数が本番環境（Cloudflare Workers / runtime.json）に正しく設定されているか確認してください。

*   **`VITE_SP_LIST_BILLING_ORDERS`**
    *   値: `c4be5492-9803-4fc6-ac7e-82d10e95ff6d`
    *   説明: コーヒー注文履歴 (`List3`) の GUID
*   **`VITE_SP_LIST_BILLING_ORDERS_SITE_RELATIVE`**
    *   値: `/sites/2`
    *   説明: `List3` が存在する SharePoint サイトの相対パス

> [!WARNING]
> これらの環境変数は本番環境設定ファイルまたは実行時の runtime env にのみ指定し、`.env` や `public/env.runtime.json`, `dist/env.runtime.json` などのリポジトリ管理下のファイルには実値をコミットしないでください。

### 2. ローカル開発環境での動作仕様について

*   ローカル標準設定（`/sites/welfare` を参照）では、`List3` の精算カラムが未整備であるため、`/billing` 画面ロード時に以下のような警告が表示されることがあります。
    *   `精算状態の永続化列が未設定です。一時的なローカル記憶（LocalStorage）で処理しています。`
*   これは、SharePoint 側のカラム不一致を検知した際の**安全な防御的フォールバック挙動**であり、プログラムの不具合ではありません。
*   本番環境で上記の `VITE_SP_LIST_BILLING_ORDERS` GUID と `/sites/2` が正しく構成されれば、物理列への読み書き（精算状態の永続化）が直接行われ、この警告は消えます。

### 3. コーヒー注文データの集計仕様 (50円フォールバック)

*   コーヒーの単価データ (`DRINK_PRICE`) が `null` もしくは未定義の場合、自動的に **50円** にフォールバックして集計・計算が行われます。
*   これは現行の業務ロジックの固定方針であり、実データ検証時にも提供済杯数 × 50円 で正しく金額が表示されることを確認しています。
