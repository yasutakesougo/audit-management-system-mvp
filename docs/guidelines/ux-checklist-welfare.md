# 確認しやすい / 入力しやすい チェックリスト（福祉現場用）

> **目的**: 新しい画面を作るとき・既存画面を改善するとき・リリース前の最終確認に使う。
> YES が多いほど「現場で使われるUI」。

---

## 🟦 ① 確認しやすいチェック（見る）

### A. 一瞬で分かるか（最重要）

- [ ] 画面を開いて **3秒以内に何の画面か分かる**
- [ ] 「今日やること」が上にある
- [ ] 重要な情報（危険・未対応）が目に入る位置にある
- [ ] 見る場所が1つに絞られている（迷わない）

### B. 探さなくていいか

- [ ] Todayや導線から **直接ここに来られる**
- [ ] 注目日・対象者がハイライトされている
- [ ] スクロールしなくても重要情報が見える
- [ ] 「どこを見ればいいか」説明なしで分かる

### C. 情報が整理されているか

- [ ] 情報が多すぎない（削れるものは削っている）
- [ ] グルーピングが直感的（例：運営・一覧）
- [ ] ラベルが分かりやすい（専門用語NG）
- [ ] 同じ意味の情報が重複していない

### D. 状態が分かるか

- [ ] 危険・注意・余裕が色やアイコンで分かる
- [ ] 空状態でも意味が分かる（例：データがありません → 正常）
- [ ] フィルタ状態が分かる
- [ ] 更新状況（最新かどうか）が分かる

---

## 🟩 ② 入力しやすいチェック（書く）

### E. すぐ入力できるか（最重要）

- [ ] 入力ボタンが迷わず押せる位置にある
- [ ] どこから入力するか一瞬で分かる
- [ ] クリック → すぐ入力画面に行ける（2タップ以内）

### F. 項目がシンプルか

- [ ] 必須項目が最小限
- [ ] 不要な入力がない
- [ ] 後から追記できる設計になっている
- [ ] 入力項目の意味が分かる（説明不要）

### G. 入力の負担が少ないか

- [ ] 初期値が入っている（日付・担当者など）
- [ ] 選択式（ドロップダウン）で済む部分が多い
- [ ] 手入力を減らしている
- [ ] 同じ入力を繰り返さなくていい

### H. ミスしにくいか

- [ ] 入力エラーが分かりやすい
- [ ] 間違えてもすぐ戻せる
- [ ] 保存前に違和感に気づける
- [ ] 途中保存できる（理想）

---

## 🟨 ③ 導線チェック（超重要）

### I. 確認 → 入力がつながっているか

- [ ] 見ている画面からそのまま入力に行ける
- [ ] 運営 → 日 → 入力 の流れが自然
- [ ] 注目日からその日の記録に飛べる
- [ ] 一度戻らなくても次の行動に行ける

### J. 迷いがないか

- [ ] 同じ機能の入口が複数ない
- [ ] 「どっちを使えばいいか」問題がない
- [ ] サイドナビがシンプル
- [ ] 画面遷移が予測できる

---

## 🟥 NGパターン（見つけたら改善）

- [ ] 同じデータが別画面にある
- [ ] 高機能だけど使い方を説明しないと分からない
- [ ] 入力前に考えることが多い
- [ ] 重要情報がスクロールしないと見えない
- [ ] フィルタが残っていて「データがない」と誤解する

---

## 🏆 合格ライン（目安）

| スコア | 判定 |
|-------|------|
| YES 80%以上 | ✅ 現場で使われる |
| YES 60〜79% | ⚠️ 改善余地あり |
| YES 59%以下 | ❌ 使われない可能性高い |

---

## 📐 適用例：Schedule 統合（PR #1121）

| カテゴリ | 評価 | 根拠 |
|---------|------|------|
| 確認しやすい | ◎ | タブ統合 + Today導線 + 注目日バナー |
| 入力しやすい | ◯ | 入力フォームへの直接遷移は今後の強化ポイント |
| 導線 | ◎ | 運営タブ → 日タブ遷移が完成 |

### 次の進化ポイント

```
判断（運営タブ） → 確認（日タブ） → 記録（入力フォーム）
```

この3ステップが自動遷移でつながると「完全体」。
