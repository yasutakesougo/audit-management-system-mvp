# 画面ワイヤー（3画面）

> 前提: [navigation-structure.md](./navigation-structure.md) のナビゲーション憲章・[ADR-005](./adr/ADR-005-isp-three-layer-separation.md) の三層モデルに基づく。
> UIラベルは統一規則に従う。「第N層」は画面上に出さない。

---

## 1. 利用者詳細 `/users/:id` — 分岐ハブ

**役割**: 全利用者共通の「操作起点」。ここで通常利用者 / IBD対象者に分岐する。

```
┌────────────────────────────────────────────────────────┐
│ ← 利用者一覧                                            │
├────────────────────────────────────────────────────────┤
│                                                        │
│  [Avatar]  氏名 (UserID)                               │
│            グループ名 / 障害支援区分                      │
│            [IBD対象者の場合のみ] 🔴 強度行動障害支援対象  │
│                                                        │
├── サマリー ────────────────────────────────────────────│
│  [ 今日の記録 ]  [ 最終記録日 ]  [ 未申し送り ]           │
│  [IBD対象者のみ] [ 見直しまで ○日 ]                      │

│  ┌─────────────┐  ┌─────────────┐                     │
│  │  📝         │  │  📨         │                     │
```

**実装上の注意**

- 現状 `buildQuickActions()` は IBD 判定未考慮。ここが次の実装ポイント

## 2. 支援計画シート詳細 `/support-planning-sheet/:id` — 第2層の中心

**役割**: IBD対象者の行動特性分析・支援設計・実施への接続。SPS を編集する唯一の場所。

```
┌────────────────────────────────────────────────────────┐
│ ← 支援計画シート一覧                                    │
│                                                [編集]  │
├── ヘッダー ────────────────────────────────────────────│
│  利用者名                                               │
│  SPS バージョン / 作成日 / 作成者                        │
│  ステータス: [行動特性アセスメント|計画作成|手順化|…]     │
│                                                        │
│  ⚠️ 見直し期限まで ○日（MonitoringCountdown）           │
│  ※ 90日超過時: 🔴 要見直し                             │
│                                                        │
├── アクションボタン ────────────────────────────────────│
│  [ ▶ 支援手順の実施 ]          ← /daily/support?planningSheetId=xxx
│  [ 🔄 見直し・PDCA ]           ← /ibd/pdca/:id        │
│                                                        │
├── タブ ────────────────────────────────────────────────│
│  [氷山分析] [支援設計] [手順スケジュール] [履歴]         │
│                                                        │
│  ── 氷山分析タブ ──────────────────────────────────   │
│  表面的な行動（observableBehaviors）                    │
│  ┌──────────────────────────────────────┐             │
│  │  · 行動観察1                          │             │
│  │  · 行動観察2                          │             │
│  └──────────────────────────────────────┘             │
│                                                        │
│  背景要因（underlyingFactors）                          │
│  ┌──────────────────────────────────────┐             │
│  │  · 背景要因1                          │             │
│  └──────────────────────────────────────┘             │
│                                                        │
│  環境調整（environmentalAdjustments）                   │
│  ┌──────────────────────────────────────┐             │
│  │  · 調整策1                            │             │
│  └──────────────────────────────────────┘             │
│                                                        │
│  良い状態の条件（positiveConditions）                   │
│  ┌──────────────────────────────────────┐             │
│  │  · 条件1                              │             │
│  └──────────────────────────────────────┘             │
│                                                        │
│  ── 支援設計タブ ──────────────────────────────────   │
│  対応方針 / 関わり方の具体策                             │
│                                                        │
│  ── 手順スケジュールタブ ───────────────────────────   │
│  場面一覧（SupportScene）                               │
│  [来所時] [食事] [活動] [場面転換] [パニック時] …       │
│  各場面 → SupportProcedureStep 一覧                    │
│                                                        │
│  ── 履歴タブ ───────────────────────────────────────  │
│  版履歴（SPSHistoryEntry）                              │
│  [インポート履歴]                                        │
│                                                        │
└────────────────────────────────────────────────────────┘
```

**実装上の注意**
- 「支援手順の実施」ボタンは `planningSheetId` を渡して `/daily/support` へ遷移
- SPS の内容編集は **このページのみ**。`/daily/support` からは変更不可（read-only）
- `currentPhase` によってタブのハイライトや次アクション推奨を変える

## 3. 支援手順の実施 `/daily/support?wizard=user` — ブリッジUI

**役割**: SPSに定義された手順を現場で実行しながら記録する。第2層→第3層の接続点。

```
┌────────────────────────────────────────────────────────┐
│ ← 戻る          支援手順の実施          [○○さん分保存] │
├── ウィザードステッパー ─────────────────────────────────│
│  ① 利用者選択  ──→  ② 支援計画選択  ──→  ③ 手順実施  │
│                                                        │
│  ── Step 1: 利用者選択（wizard=user）─────────────────│
│  ┌──────────────────────────────────────┐             │
│  │  🔍 利用者を検索...                   │             │
│  ├──────────────────────────────────────┤             │
│  │  ○ ○○さん  [IBD対象 🔴]             │             │
│  │  ○ △△さん  [IBD対象 🔴]             │             │
│  └──────────────────────────────────────┘             │
│  ※ IBD対象者のみ表示（非対象者はこの画面に入れない）     │
│                                            [ 次へ → ] │
│                                                        │
│  ── Step 2: 支援計画選択 ───────────────────────────  │
│  ┌──────────────────────────────────────┐             │
│  │  支援計画シート v3  (2026-01-15〜)    │  [選択]    │
│  │  支援計画シート v2  (2025-10-01〜)    │            │
│  └──────────────────────────────────────┘             │
│  ※ planningSheetId 指定で自動選択済みの場合はスキップ   │
│                                  [ ← 戻る ] [ 次へ → ]│
│                                                        │
│  ── Step 3: 手順実施 ───────────────────────────────  │
│  ┌── ヘッダー ──────────────────────────────────────┐ │
│  │  ○○さん  |  記録日: 2026-04-09  |  記録者: ___   │ │
│  │  ⚠️ 見直しまで 12日（MonitoringCountdown）        │ │
│  └───────────────────────────────────────────────────┘ │
│                                                        │
│  場面タブ: [来所時] [食事] [活動] [場面転換] …           │
│                                                        │
│  ┌── 手順ステップ一覧（schedule）──────────────────┐  │
│  │  ☑ Step 1: ○○を確認する          [済]          │  │
│  │  ☐ Step 2: △△を実施する          [未]          │  │
│  │  ☐ Step 3: □□を記録する          [未]          │  │
│  └───────────────────────────────────────────────────┘ │
│                                                        │
│  ┌── 選択ステップの記録入力 ──────────────────────┐  │
│  │  手順通り実施: [○ はい] [× いいえ（理由記入）]  │  │
│  │  利用者の様子: ___________________________       │  │
│  │  特記事項:     ___________________________       │  │
│  └───────────────────────────────────────────────────┘ │
│                                                        │
│  進捗: ████░░░ 1/3 完了                               │
│                                                        │
│  [ 下書き保存 ]                  [ ○人分保存 → 送信 ] │
│                                                        │
│  ── 完了後の導線 ────────────────────────────────── │
│  [ 🔄 見直し・PDCA へ ]  [ ← 支援計画シートに戻る ]    │
│                                                        │
└────────────────────────────────────────────────────────┘
```

**実装上の注意**
- Step3 は SPS を **read-only** で参照。このページからの SPS 変更は禁止
- `filledStepIds` で未記入ステップをハイライト。未記入フィルタ切替あり
- 完了後の PDCA 導線は `IcebergPdcaUrl` を使用（既存実装済み）
- IBD 非対象者が直接 URL アクセスした場合は `/daily/table` へリダイレクト

## 3画面の接続フロー

```
利用者詳細 (/users/:id)
 │
 ├── [IBD対象者] 支援計画シートボタン
 │    └──→ 支援計画シート詳細 (/support-planning-sheet/:id)
 │               │
 │               ├── [ ▶ 支援手順の実施 ] ?planningSheetId=xxx
 │               │    └──→ 支援手順の実施 (/daily/support)
 │               │               │
 │               │               └── [ 見直し・PDCA ]
 │               │                    └──→ PDCA画面
 │               │                         └── SPS改訂 → ISP見直し
 │               └── [ 見直し・PDCA ]
 │                    └──→ PDCA画面
 │
 └── [全利用者] 日々の記録ボタン
      └──→ ケース記録 (/daily/table)
```
