# PoC運用サポート資料集

## 🔥 当日から使える想定FAQ

### 📱 基本操作について

**Q1: ログインができません**
**A1**:
- ブラウザのキャッシュをクリア
- ID・パスワードの再確認
- IT担当者（XXX）にお声がけください

**Q2: 申し送りの入力方法がわかりません**
**A2**:
1. 申し送りページを開く
2. 「新規作成」ボタンをクリック
3. 必要項目を入力（タイトルは自動生成されます）
4. 「保存」をクリック

**Q3: 間違って入力してしまいました**
**A3**:
- いつでも編集・削除可能です
- 「編集」ボタンから修正してください
- 不安な場合は管理者にご相談を

**Q4: 入力中にエラーが出ました**
**A4**:
- ページを更新（F5キー）
- 再度ログインしてみる
- 改善しない場合は速やかに報告

### 🕐 業務フローについて

**Q5: いつ入力すればいいですか？**
**A5**:
- 気づいた時にすぐ入力（推奨）
- 最低でも勤務終了前に1日の内容をまとめて入力
- 朝会・夕会前の確認も効果的

**Q6: 紙のメモは併用してもいいですか？**
**A6**:
- PoC期間中は併用OK
- システムに慣れたら徐々にシステムメインに
- 重要な内容は必ずシステムにも入力を

**Q7: 朝会・夕会での使い方は？**
**A7**:
- 司会者：画面のサマリーを読み上げ
- 参加者：追加情報があれば補足
- 重要案件のアラート表示を必ず確認

### 📝 入力内容について

**Q8: どこまで詳しく書けばいいですか？**
**A8**:
- ポイントだけでOK（1-2行程度）
- 時間・場所・対象者・内容を簡潔に
- 例：「14:00頃 A様 転倒リスク高 要注意」

**Q9: 個人情報の入力は大丈夫ですか？**
**A9**:
- 既存システムと同等のセキュリティレベル
- 必要最小限の情報のみ入力
- 利用者名は愛称・イニシャルでも可

**Q10: 他の人の入力内容は見られますか？**
**A10**:
- 同じ施設の職員のみ閲覧可能
- 情報共有の目的での使用
- プライバシー配慮は継続

### ⚡ トラブルシューティング

**Q11: 動作が遅いです**
**A11**:
- 他のアプリケーションを閉じる
- Wi-Fi接続の確認
- ブラウザの再起動

**Q12: データが消えました**
**A12**:
- 入力中は定期的に「保存」ボタンを押す
- データ復旧が可能な場合があるので速やかに報告
- バックアップ機能で対応

**Q13: スマホ・タブレットでも使えますか？**
**A13**:
- 基本的に対応済み
- 画面サイズに応じて表示が最適化
- 操作方法は基本的に同じ

## 🎯 振り返りミーティング運営ガイド

### 📋 第1回振り返り（第1週末）

#### 🕐 所要時間: 30分

#### 📝 議題
1. **使用状況の確認（10分）**
   - 入力件数・参加率の報告
   - 基本操作の習得状況
   - 発生したトラブル・質問

2. **効果の実感確認（10分）**
   - 便利に感じた機能
   - 業務への影響（ポジティブ・ネガティブ）
   - 朝会・夕会での活用度

3. **改善点の整理（10分）**
   - 操作上の困りごと
   - 機能への要望
   - 第2週に向けた調整事項

#### ✅ 確認フォーマット
```
【第1回振り返り - 参加者アンケート】

1. システムの操作は慣れましたか？
   □ 十分慣れた  □ だいぶ慣れた  □ まだ難しい

2. 申し送り業務に変化を感じますか？
   □ 大変便利  □ 少し便利  □ あまり変わらない  □ 使いにくい

3. 朝会・夕会での活用はいかがですか？
   □ とても有効  □ 有効  □ まだよくわからない

4. 改善してほしい点があれば教えてください
   自由記載：

5. 第2週も継続して使いたいですか？
   □ ぜひ使いたい  □ 使いたい  □ どちらでもよい  □ 使いたくない
```

### 📋 最終評価ミーティング（第2週末）

#### 🕐 所要時間: 60分

#### 📝 議題
1. **数値効果の確認（20分）**
   - 入力件数・参加率の最終結果
   - 時間短縮効果の測定結果
   - 見落とし防止効果の確認

2. **職員満足度の評価（20分）**
   - アンケート結果の共有
   - 具体的な改善効果の事例紹介
   - 継続使用への意向確認

3. **今後の方針決定（20分）**
   - 本格導入の可否判断
   - 改善が必要な機能の整理
   - 導入スケジュールの検討

#### 📊 評価基準
**A評価（即座に本格導入）**:
- 全項目で目標達成
- 職員満足度80%以上
- 継続使用希望90%以上

**B評価（改善後に本格導入）**:
- 主要項目で目標達成
- 職員満足度60%以上
- 具体的改善点が明確

**C評価（再検討が必要）**:
- 目標未達成項目が多数
- 職員満足度50%未満
- システム変更が必要

## 🚨 緊急時対応マニュアル

### 🔧 システムトラブル対応

#### レベル1: 個人の操作問題
**症状**: 特定の人だけが操作できない
**対応**:
1. ペアサポート（隣の人に操作を見てもらう）
2. 操作マニュアルの再確認
3. 管理者による個別サポート

#### レベル2: 機能の不具合
**症状**: 特定の機能が動かない
**対応**:
1. 該当機能の使用を一時停止
2. 代替手段での業務継続
3. IT担当者への報告・対応依頼

#### レベル3: システム全体停止
**症状**: 全員がアクセスできない
**対応**:
1. 即座に紙ベースでの業務継続
2. 緊急連絡網での報告
3. 復旧見込み時間の確認・共有

### 🏥 業務継続対応

#### 申し送り業務
**通常時**: システム入力
**緊急時**: 紙メモ + 口頭伝達（従来方式）
**復旧後**: 紙メモ内容のシステム入力

#### 朝会・夕会
**通常時**: ミニサマリー画面表示
**緊急時**: 各自の記憶・メモによる報告
**復旧後**: システム確認での補足

#### 日次記録
**通常時**: 自動下書き機能活用
**緊急時**: 手動での特記事項入力
**復旧後**: システム連動の再確認
