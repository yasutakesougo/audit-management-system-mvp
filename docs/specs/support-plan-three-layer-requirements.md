# 支援計画系ドメインの基本設計方針

> **ADR-005 準拠** | **イソカツシステム固有要件**  
> 作成日: 2026-03-12

## 目的

イソカツシステムでは、障害福祉サービスにおける支援計画関連情報を、制度適合性・監査耐性・現場再現性の3点を満たす形で管理する。

そのため、以下の3層を明確に分離して設計する。

1. 個別支援計画（ISP）
2. 支援計画シート
3. 支援手順書兼記録

## 基本方針

### 1. 個別支援計画（ISP）は上位計画として扱う

個別支援計画は、利用者本人の意向、家族等の意向、総合的支援方針、生活全般の課題、長期目標、短期目標、達成時期、留意事項等を定める法定の中核文書として管理する。

また、同意取得、交付、モニタリング、見直し履歴を追跡可能とする。

### 2. 支援計画シートは中間設計文書として扱う

支援計画シートは、ISPの目標や課題を現場で再現可能な支援方針へ落とし込むための中間レイヤーとする。

行動観察、情報収集、分析・仮説、支援課題、環境調整、関わり方の具体策等を記録し、版管理対象とする。

### 3. 支援手順書兼記録は現場実装ログとして扱う

支援手順書兼記録は、サービス提供時に使用する具体手順と実施ログを兼ねる文書とする。

時間帯、活動、支援手順、実施チェック、利用者の様子、特記事項、連絡事項等を記録し、再アセスメントの根拠として利用可能にする。

## 三層関係

- 1件のISPに対して複数の支援計画シートを紐づけ可能とする
- 1件の支援計画シートに対して複数の支援手順書兼記録を紐づけ可能とする
- 支援手順書兼記録から、根拠となる支援計画シートおよびISPへ遡及可能とする
- 利用者単位で、上位計画 → 支援設計 → 実施記録 を一貫して参照可能とする

## 運用フロー要件

### ISP

以下の状態を追跡可能とする。

| # | 状態 | 説明 |
|---|---|---|
| 1 | アセスメント | 本人・環境の情報収集 |
| 2 | 原案作成 | ISP ドラフト作成 |
| 3 | 会議 | 支援会議での検討 |
| 4 | 説明 | 本人・家族への説明 |
| 5 | 同意取得 | 署名・同意の記録 |
| 6 | 交付 | 計画書の交付 |
| 7 | 実施 | サービス提供中 |
| 8 | モニタリング | 定期的な評価 |
| 9 | 見直し | 計画の改訂 |

### 支援計画シート

以下の運用を追跡可能とする。

| # | 状態 | 説明 |
|---|---|---|
| 1 | 行動特性アセスメント | 行動観察・情報収集 |
| 2 | 仮説整理 | 氷山モデル分析 |
| 3 | 支援計画作成 | 環境調整・関わり方の設計 |
| 4 | 手順化 | 場面別手順書への落とし込み |
| 5 | 再アセスメント | 実施記録に基づく再評価 |
| 6 | 改訂 | 支援計画の更新 |

### 支援手順書兼記録

以下の要素を記録可能とする。

- 予定された手順
- 実施の有無
- 実施時の様子
- 特記事項
- 連絡事項
- 実施者
- 実施日時

## 監査証跡要件

すべての文書レイヤーにおいて、以下を保持する。

- 作成者
- 更新者
- 作成日時
- 更新日時
- 版番号または履歴番号
- 状態
- 関連する親文書ID

ISPについては追加で以下を保持する。

- 同意取得日時
- 交付日時
- モニタリング実施日時
- 見直し予定日
- 見直し実施日

## 設計上の禁止事項

- ISPと支援計画シートを単一データとして統合しない
- 支援計画シートをメモ欄や補足欄に矮小化しない
- 支援手順書兼記録を単なる日報として扱わない
- 同意・交付・モニタリング・見直し履歴を省略しない
- 版管理なしの上書き更新を行わない

## 既存画面との整合方針

| 既存画面 | 三層での位置づけ | 方針 |
|---|---|---|
| `SupportPlanGuidePage` | 第1層 ISP | ISP 作成・編集の中心画面として維持 |
| `ISPComparisonEditorPage` | 第1層 ISP | 比較・更新エディタとして維持 |
| `IcebergAnalysisPage` | 第2層 支援計画シート | 氷山モデル分析の入口 |
| `SupportStepMasterPage` | 第2層 支援計画シート | 手順テンプレートマスタ |
| `SupportActivityMasterPage` | 第2層 支援計画シート | 活動テンプレートマスタ |
| `TimeFlowSupportRecordPage` | 第3層 支援手順書兼記録 | 時系列の実施記録 |
| `TimeBasedSupportRecordPage` | 第3層 支援手順書兼記録 | 時間帯別の実施ログ |
| `SupportRecordPage` | 第3層 支援手順書兼記録 | 19項目の支援手順実施状況 |

既存の Daily / Support Record 系画面は、原則として第3層（支援手順書兼記録）として再整理する。
SupportPlanGuidePage 等の計画系画面は、第1層または第2層の責務へ分離再配置する。
