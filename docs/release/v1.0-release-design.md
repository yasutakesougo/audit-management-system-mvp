## 1. Release Vision

### v1.0 の定義

> **v1.0 は「1 つの事業所で 3 ヶ月運用できる」状態**

これは「全機能の完成」ではなく、以下の 3 条件を満たすことを意味する。

| # | 条件 | 意味 |
|---|------|------|
| 1 | **日次ループが回る** | Today → Daily → Handoff が毎日使われる |
| 2 | **月次ループが回る** | Monitor → Proposal → Execution が 90 日で 1 周する |
| 3 | **知識が蓄積される** | 採用/却下の履歴が残り、後から追跡できる |

### v1.0 ではないもの

| 含まない | 理由 |
|---------|------|
| マルチ事業所対応 | Phase 5 の領域 |
| AI 提案ランキング | Phase 4 の領域 |
| API / Webhook | Phase 5 の領域 |
| カスタムルールエンジン | Phase 5 の領域 |

### Quality Gate

v1.0 リリースに必要な品質基準。

| 指標 | 現在 | v1.0 目標 | 判定 |
|------|------|----------|------|
| **Unit Test Coverage (Lines)** | 44%+ | ≥ 50% | 🟡 |
| **Unit Test Coverage (Functions)** | 38%+ | ≥ 45% | 🟡 |
| **TypeCheck** | Pass | Pass | ✅ |
| **Lint** | Pass | Pass | ✅ |
| **Lighthouse Performance** | — | ≥ 70 | 🟡 |
| **Lighthouse Accessibility** | — | ≥ 80 | 🟡 |
| **E2E Smoke Test** | 部分的 | Critical Path 100% | 🟡 |
| **Bundle Size** | — | ≤ 2MB (gzip) | 🟡 |

### Critical Path テスト

v1.0 で E2E テストが 100% 必要なパス。

```
1. ログイン → Today 画面表示
2. Today → Daily 記録入力 → 保存
3. Handoff → 申し送り作成 → 保存
4. Planning Sheet → Assessment 取込 (Bridge 1)
5. Planning Sheet → Monitoring 取込 (Bridge 3)
6. ABC Record → 記録作成 → 保存
7. 出欠管理 → 出欠記録 → 保存
```

### Domain Logic テスト

`src/domain/` 内の純粋関数は **100% テストカバレッジ** を目指す。

| モジュール | 現在 | 目標 |
|-----------|------|------|

## 4. Data Strategy

### 現在のアーキテクチャ

```
開発/デモ:   LocalStorage  ← 全データ
本番:        SharePoint    ← ISP 三層 + 利用者マスタ
認証:        Firebase/MSAL ← ログイン
```

### v1.0 のデータ配置

| データ | 開発 | v1.0 本番 | 備考 |
|--------|------|-----------|------|
| ISP (L1) | localStorage | **SharePoint** | ✅ SharePoint adapter 完成済 |
| 支援計画 (L2) | localStorage | **SharePoint** | ✅ SharePoint adapter 完成済 |
| 手順書 (L3) | localStorage | **SharePoint** | ✅ SharePoint adapter 完成済 |
| Daily Record | localStorage | **SharePoint** | ✅ SharePoint adapter 完成済 |
| Handoff | localStorage | **SharePoint** | ✅ SharePoint adapter 完成済 |
| 利用者マスタ | localStorage | **SharePoint** | ✅ SharePoint adapter 完成済 |
| 出欠記録 | localStorage | **SharePoint** | SharePoint adapter 必要 |
| ABC Record | **localStorage** | **localStorage** | v1.0 はローカル |
| Evidence Link | **localStorage** | **localStorage** | v1.0 はローカル |
| 身体拘束 | **localStorage** | **localStorage** | v1.0 はローカル |
| 危機対応 | **localStorage** | **localStorage** | v1.0 はローカル |
| Compliance | **localStorage** | **localStorage** | v1.0 はローカル |
| ISP Decision | InMemory | **SharePoint** | SharePoint adapter 完成済 |

### マイグレーション戦略

```
Phase A: localStorage のみで運用開始（安全）
  ↓
Phase B: ISP 三層を SharePoint に移行（SharePoint adapter 済）
  ↓
Phase C: Daily / Handoff を SharePoint に移行
  ↓
Phase D: 残りのデータを段階的に移行
```

### バックアップ戦略

| データ | バックアップ方法 | RPO（復旧点目標） |
|--------|----------------|------------------|
| SharePoint 上データ | Microsoft 365 標準 | 自動 |
| localStorage データ | エクスポート機能（手動） | ユーザー次第 |
| 設定・マスタ | Git 管理 | push 時 |

## 5. Deployment Architecture

### v1.0 の構成

```
┌───────────────────────────────────────────────────────┐
│  Browser (SPA)                                        │
│                                                       │
│  React 18 + Vite                                      │
│  ├── MUI (UI)                                         │
│  ├── React Query (Server State)                       │
│  ├── Zustand (Client State)                           │
│  └── localStorage (一部永続化)                        │
│                                                       │
└────────────┬──────────────────┬───────────────────────┘
             │                  │
             ▼                  ▼
┌──────────────────┐  ┌──────────────────────────────┐
│  Firebase Auth   │  │  SharePoint Online REST API   │
│  (or MSAL)       │  │                                │
│                  │  │  ISP / Planning / Procedure    │
│  認証            │  │  Daily / Handoff / Users       │
└──────────────────┘  └────────────┬───────────────────┘
                                   │
                                   ▼
                      ┌────────────────────────┐
                      │  Cloudflare Worker     │
                      │  (Token Exchange)      │
                      └────────────────────────┘
```

### 配信

| 方式 | 詳細 |
|------|------|
| **ホスティング** | Azure Static Web Apps or Cloudflare Pages |
| **ビルド** | `npm run build` → `/dist` |
| **CI/CD** | GitHub Actions → Preview Deploy → Production Deploy |
| **URL** | `https://support-ops.example.com` |

### 環境分離

| 環境 | 用途 | データ |
|------|------|--------|
| `development` | ローカル開発 | localStorage (モック) |
| `preview` | PR プレビュー | localStorage (モック) |
| `staging` | 結合テスト | SharePoint (テストサイト) |
| `production` | 本番運用 | SharePoint (本番サイト) |

## 7. Governance

### バージョニング

[Semantic Versioning](https://semver.org/) に従う。

```
MAJOR.MINOR.PATCH

例:
1.0.0  ← v1.0 初回リリース
1.1.0  ← Phase 3 の ProposalCard 統合追加
1.2.0  ← 新分析ルール追加
2.0.0  ← マルチ事業所対応（破壊的変更）
```

### リリースサイクル

```
Main Branch
  │
  ├── feature branch → PR → merge → main
  │
  ├── 月次リリース（minor: 1.x.0）
  │     → 新機能・改善
  │
  ├── 随時リリース（patch: 1.0.x）
  │     → バグ修正・セキュリティ
  │
  └── 年次リリース（major: x.0.0）
        → 破壊的変更
```

### Issue 管理

| ラベル | 用途 |
|--------|------|
| `type: feature` | 新機能 |
| `type: bug` | バグ修正 |
| `type: docs` | ドキュメント |
| `type: refactor` | リファクタリング |
| `priority: P0` | 安全管理（即時対応） |
| `priority: P1` | 業務阻害（当日対応） |
| `priority: P2` | 改善希望（計画的対応） |
| `principle: N` | 設計原則 N に関連 |
| `phase: N` | ロードマップ Phase N |

### ADR プロセス

```
1. 設計上の重要な判断が必要になる
2. docs/adr/ に ADR-NNN を Draft として作成
3. PR レビューで議論
4. マージ時に Accepted に変更
5. ADR Index (README.md) を更新
```

## 8. Risk Assessment

### 技術リスク

| リスク | 影響 | 確率 | 軽減策 |
|--------|------|------|--------|
| SharePoint API の rate limit | データ取得遅延 | 中 | spClient.ts のリトライ + キャッシュ |
| localStorage の容量上限 (5-10MB) | データ消失 | 低 | エクスポート機能 + 定期警告 |
| Bundle Size の肥大化 | 初期表示遅延 | 中 | Code Splitting + Lazy Loading |
| MSAL トークン期限切れ | 認証エラー | 低 | サイレントリフレッシュ実装済 |

### 運用リスク

| リスク | 影響 | 確率 | 軽減策 |
|--------|------|------|--------|
| 現場が使わない | プロジェクト失敗 | 中 | 原則 1（現場導線主体）で設計済 |
| 提案を無視する | PDCA 形骸化 | 中 | 原則 1（控えめ表示） + 非侵入的 UI |
| 入力負担の増大 | 現場離脱 | 低 | 原則 2（入力最小化） |
| データ移行失敗 | 運用中断 | 低 | 段階的移行 + ロールバック手順 |

### セキュリティリスク

| リスク | 影響 | 確率 | 軽減策 |
|--------|------|------|--------|
| localStorage に個人情報 | 情報漏洩 | 中 | 機密データは SharePoint のみに保存 |
| CSRF / XSS | データ改ざん | 低 | React のデフォルト保護 + CSP 設定 |
| SharePoint の権限設定ミス | 不正アクセス | 低 | provision/schema.xml で管理 |
