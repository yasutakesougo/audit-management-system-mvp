# Kiosk Quick Link 観測期間ルール

## 1. 目的
- `KioskQuickLinks` の利用状況を `target = link.id` 単位で観測する。
- 観測の主キーはラベルではなく `link.id` を使う。
- 文言改善・並び順調整・補助説明追加の判断材料として使う。

## 2. 観測期間
- 標準観測期間は **直近7日** とする。
- 理由:
  - 変化を早めに拾える。
  - 現場運用の週次リズムに合わせやすい。
- 補助観測として **直近14日** を再確認窓に使ってよい。
- 判定ルール:
  - 一次判定: 直近7日
  - 再判定: 必要に応じて直近14日

## 3. 観測対象
- イベント: `ux_navigate_from_today`
- 集計単位: `target`
- 対象 `link.id`:
  - `schedule`
  - `handoff`
  - `minutes`
  - `room`
  - `briefing`

## 4. 低利用の判定基準（暫定）
- **低利用候補**: 直近7日で `target` ごとの件数が **0〜1件**
- **要観察**: 直近7日で **2〜3件**
- **通常利用**: 直近7日で **4件以上**
- 補足:
  - 本基準は暫定であり、実データ蓄積後に見直す。
  - 必要に応じて絶対件数に加え、全体比率を追加する。

## 5. 改善判断ルール
- 低利用候補になっただけで即削除しない。
- 改善優先順:
  1. サブテキスト追加
  2. 文言見直し
  3. 並び順見直し
  4. 必要なら導線統合を検討
- `briefing` と `minutes` のような意味が近い導線は、低利用時の優先レビュー対象とする。

## 6. 確認場所
- Telemetry Dashboard の Kiosk UX セクションを正とする。
- `target` 単位の件数表示を観測結果として扱う。
- 必要に応じて nightly への展開を将来対応として扱う。

## 7. レビュー用の短い要約
`KioskQuickLinks の利用状況は target = link.id を主キーとして直近7日を標準観測期間に集計し、0〜1件を低利用候補として文言・補助説明・並び順改善の対象とする。`
