# TodayOps 試運用レビュー会テンプレート（30分版）

> **目的**: 感想大会ではなく改善会議にする
>
> 試運用の結果を **数字と事実** で振り返り、
> **次にやること / やらないこと** を決めて終わる。

---

## 前提

| 項目 | 値 |
|------|-----|
| 所要時間 | 30分（厳守） |
| 参加者 | 現場職員 2〜3名 + 開発 1名 |
| 開催時期 | `VITE_FEATURE_TODAY_OPS=1` 有効化から 1 週間後 |
| 準備物 | 観測データ（下記）、このテンプレート |

---

## タイムライン

| 時間 | 内容 | 担当 |
|------|------|------|
| 0:00〜0:03 | **① 数字の確認** | 開発 |
| 0:03〜0:10 | **② 3つの問い** | 現場職員 |
| 0:10〜0:20 | **③ 困った場面** | 現場職員 |
| 0:20〜0:25 | **④ やる / やらないの判断** | 全員 |
| 0:25〜0:30 | **⑤ 次のアクション確定** | 開発 |

---

## ① 数字の確認（3分）

開発側が事前に用意して読み上げる。

| 指標 | 目標 | 実績 | 判定 |
|------|------|------|------|
| `/today` 日次到達率 | 80%以上 | ＿＿% | ⭕ / ❌ |
| `/dashboard` 戻り率 | 20%以下 | ＿＿% | ⭕ / ❌ |
| 初回タップ先 TOP3 | — | 1.＿＿ 2.＿＿ 3.＿＿ | — |
| 苦情・問い合わせ件数 | 0件 | ＿＿件 | ⭕ / ❌ |

### データソース

```ts
// 事前に仕込むログ（試運用前に実装推奨）
auditLog('today.page_view');
auditLog('today.first_click', { target });
auditLog('today.dashboard_return');
```

> [!TIP]
> ログが無い場合は、職員への聞き取り（「朝一番にどこを開きましたか？」）で代用可。

### 問い3: この画面が無くなったら困りますか？

|---|------|----------------|-----------|------------|

## ④ やる / やらないの判断（5分）

③で出た問題を **即判断** する。

| 問題 | 判定 | 理由 |
|------|------|------|
| ＿＿＿ | ⭕やる / ❌やらない / 🟡保留 | ＿＿＿ |
| ＿＿＿ | ⭕やる / ❌やらない / 🟡保留 | ＿＿＿ |
| ＿＿＿ | ⭕やる / ❌やらない / 🟡保留 | ＿＿＿ |

### 判定基準

| 判定 | 基準 |
|------|------|
| ⭕ やる | 2人以上が困った + 実装 1日以内 |
| 🟡 保留 | 1人だけ困った or 実装 2日以上 |
| ❌ やらない | 操作に慣れれば解消する + 本質的な価値に影響しない |

### サイドメニュー改善候補（事前準備済み）

試運用で該当する苦情が出たら、以下を ⭕ に変更：

| # | 候補 | 試運用で出たか |
|---|------|:---:|

| 2 | グループ項目数の整理（8→5） | ⬜ |
| 3 | 「記録を参照」見出し表記の整合 | ⬜ |

---

## 会議の禁止事項

| ❌ やらない | 理由 |
|-----------|------|
| 新機能のアイデア出し | この会は「改善」だけ。新機能は別会議 |
| 全員の感想を一巡 | 3つの問いに集中。感想は無限に出る |
| 技術的な設計議論 | この会は「現場の事実」を集める場 |
| 30分超過 | 超えたら次回に持ち越し |
