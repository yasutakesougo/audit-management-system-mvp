# 3画面責務 — 配置判断クイックリファレンス

> 新機能を「Dashboard / Today / Handoff Timeline」のどこに置くか、1画面で決める早見表。
> 詳細は [screen-responsibility-map.md](./screen-responsibility-map.md) を参照。

---

## 1. 一言定義

| 画面 | 一言 | キーワード |
|------|------|-----------|
| **Dashboard** `/dashboard` | **読んで判断**する画面 | 俯瞰・統計・傾向・アラート |
| **Today** `/today` | **やって消す**画面 | 今日・共有・未処理ゼロ・実行 |
| **Handoff** `/handoff-timeline` | **書いて追跡**する画面 | 申し送り・状態遷移・会議クローズ・監査 |

---

## 2. 置く / 置かない

| | Dashboard | Today | Handoff |
|---|:-:|:-:|:-:|
| 集計・傾向グラフ | ✅ | ❌ | ❌ |
| 管理者向けアラート | ✅ | ❌ | ❌ |
| 朝会/夕会の統計振り返り | ✅ | ❌ | ❌ |
| 未処理タスク CTA | ❌ | ✅ | ❌ |
| 出席・業務体制 | ❌ | ✅ | ❌ |
| ブリーフィング（今日の共有事項） | ❌ | ✅ | ❌ |
| 送迎進捗 | ❌ | ✅ | ❌ |
| 申し送り作成・編集 | ❌ | ❌ | ✅ |
| ステータス遷移・会議モード | ❌ | ❌ | ✅ |
| コメント・監査ログ | ❌ | ❌ | ✅ |
| 申し送りの簡易表示 | ❌ | ⚠️ 読取のみ | ✅ |

⚠️ = consumer として簡易表示のみ。操作は Handoff に委譲。

---

## 3. 迷ったらこのフロー

```
新しい機能 → ユーザーが書き込む？
  ├─ No → 過去データの分析？ → Yes → 📊 Dashboard
  │                          → No  → 今日中に見る？ → Yes → 📋 Today
  │                                                  → No  → 専用ページ
  └─ Yes → 申し送りに関係ある？
       ├─ Yes → 📝 Handoff Timeline
       └─ No  → 今日中に完了すべき？ → Yes → 📋 Today
                                      → No  → 専用ページ*
```

*専用ページの例: 設定、月次管理、個別支援計画など
---

## 4. よくある迷いパターン

| 迷い | 答え | 一言理由 |
|------|------|---------|
| 未記録アラート | **Today** | 今日中に潰すタスク |
| 今週の記録率グラフ | **Dashboard** | 俯瞰・判断 |
| 気になった点をメモ | **Handoff** | 追跡・状態管理が必要 |
| 職員の在席確認 | **Today** | 今日の業務体制 |
| 先週の申し送り傾向 | **Dashboard** | 統計・分析 |
| 朝会で昨日分を振り返る | **Handoff** | dayScope='yesterday' |
| 申し送りの「件数だけ」Today に出したい | **Today** ⚠️ | 読み取りのみ、操作は Handoff へ |
