# 環境フラグ運用メモ

## localStorage override 方針

- `prod` / `staging` では原則 `localStorage["feature:*"]` を使用しない。
- パイロット中に一時的に override を使う場合は、端末・担当者・ON にしたフラグを記録しておく。
- フェーズ切替前には、対象ブラウザの `feature:*` キーを全削除するチェックリストを用意する。

## フェーズ切替の責任分担

- `.env.*` の変更権限はシステム担当に限定する。
- 現場からの要望は「Phase1→2 のどのフラグを変えるか」といった形で上げてもらい、設定変更は運用側で反映する。
- 変更後は `docs/design/schedules-rollout.md` のフラグマトリクスと突き合わせ、意図せぬ露出がないか確認する。
