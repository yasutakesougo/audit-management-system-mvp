# ブリッジ機能 運用ガイド

> **対象読者**: 支援計画を作成・編集する職員、管理者
>
> **最終更新**: 2026-03-14

---

## 1. この機能でできること

### アセスメント → 支援計画シート

アセスメント（ICF 分類・感覚プロファイル）や特性アンケート（保護者回答）の結果を、
支援計画シートの各項目に **安全に取り込む** ことができます。

| 取込元 | 取込先 | 内容 |
|---|---|---|
| ICF 分類（身体機能・活動・環境因子） | 行動観察 / 収集情報 | 分類ラベル＋所見を追記 |
| 感覚プロファイル | 感覚トリガー（情報収集） | 過敏・低反応を自動検出 |
| 分析タグ | 医療フラグ（情報収集） | 関連タグを追加 |
| 特性アンケート | 行動観察 / 収集情報 | 集約された特性情報を追記 |

### 支援計画シート → 手順書兼記録

支援計画シートの方針・具体策を、日次の手順ステップに変換できます。

| 取込元 | 取込先 | 手順の種類 |
|---|---|---|
| 対応方針 | 手順ステップ（activity=支援方針） | ★ 重要ポイント |
| 具体策 | 手順ステップ（activity=具体的対応） | 通常ステップ |
| 環境調整 | 手順ステップ（activity=環境調整） | 留意点 |
| 感覚トリガー / 医療フラグ | 全ステップ共通注記 | 補助情報 |

## 2. できないこと（制限事項）

| 制限 | 理由 |
|---|---|
| **上書き取込** | 既存データを保護するため、追記マージのみ対応 |
| **部分取込** | 現時点では全項目一括取込のみ（将来対応予定） |
| **取込の取消（Undo）** | 取込後の編集は手動で行ってください |
| **他利用者からのデータ転用** | 取込元は同一利用者のアセスメントのみ |
| **自動保存** | 取込後は「保存」ボタンを押すまでフォームに反映されているだけです |

## 3. 取込の流れ

### アセスメント → 支援計画シート

```
① 支援計画シートを「編集モード」にする
② 「アセスメントから取込」ボタンをクリック
③ ダイアログでプレビューを確認
   - どの項目に何が入るか
   - 変換理由
④ 「取り込む」をクリック
⑤ フォームに値が入る（この時点ではまだ保存されていません）
⑥ 内容を確認・必要に応じて編集
⑦ 「保存」ボタンで確定
```

### 支援計画シート → 手順書兼記録

```
① 日次支援画面で手順パネルを表示
② 「計画シートから取込」ボタンをクリック
③ ダイアログでプレビューを確認
   - 追加されるステップ数
   - 方針 / 具体策 / 環境調整 の分類
   - 全ステップ共通注記の内容
④ 「取り込む」をクリック
⑤ 手順一覧にステップが追加される
```

## 4. provenance（出典）の見方

### フィールド横バッジ

支援計画シートの編集画面で、各入力欄の直下に表示されます。

| バッジの色 | 意味 |
|---|---|
| 🔵 **ICF** | ICF 分類アイテム由来 |
| 🟢 **アンケ** | 特性アンケート由来 |
| 🔵 **感覚** | 感覚プロファイル由来 |
| 🟣 **タグ** | 分析タグ由来 |
| ⚡ **計画** | 支援計画シート由来（手順書側で表示） |

**ホバーで詳細が見えます**: バッジにマウスを合わせると、
「なぜこの値が入ったか」の変換理由が表示されます。

### 出典パネル（ProvenancePanel）

編集モードで取込済みの項目がある場合、タブの上に出典パネルが表示されます。
すべての出典情報を一覧で確認できます。

## 5. `[📋 計画]` バッジの意味

手順書兼記録の手順一覧で表示される `[📋 計画]` バッジは、
**その手順が支援計画シートから取り込まれたもの**であることを示します。

- ホバーすると「支援計画シートから取込された手順」と表示されます
- バッジのない手順は、職員が手入力したものです
- バッジがあっても、内容は自由に編集できます

## 6. 取込履歴の見方

支援計画シート画面に「取込履歴」パネルが表示されます。

### 一覧で見える情報

| 項目 | 内容 |
|---|---|
| 日時 | 取込を実行した日時 |
| 実行者 | 取込を実行した職員名 |
| モード | 「アセスメントのみ」or「アセスメント＋特性アンケート」 |
| 変更項目数 | 何項目が変更されたか |

### 展開で見える情報

| 項目 | 内容 |
|---|---|
| サマリー | 取込内容の要約テキスト |
| 変更フィールド | 具体的にどのフィールドが変わったか |
| 変換根拠 | 各出典からの変換理由 |

## 7. 再取込したときの挙動

### 同一データの再取込

**重複しません**。すでに取り込まれている内容と同じデータを再取込すると、
「追加項目はありません」と表示されます。

### 元データが更新された場合

アセスメントや特性アンケートが更新された後に再取込すると、
**新しく追加された部分のみ**が追記されます。
既存の取込済みデータは変更されません。

### 手動で編集した内容

取込後に職員が手動で編集した内容は、再取込で上書きされません。
取込はあくまで「追記」であり、既存テキストを消すことはありません。

## 8. 職員が手で確認・編集すべき点

> **重要**: 取込機能はあくまで「下書きの補助」です。
> 最終的な計画内容の責任は作成職員にあります。

### 必ず確認してほしいこと

| 確認項目 | なぜ必要か |
|---|---|
| **取り込まれた文章の適切さ** | 自動変換された文章が、この利用者に合っているか |
| **不要な項目の削除** | すべての取込項目が必要とは限りません |
| **表現の調整** | 専門用語を現場に合った表現に直す |
| **追加の記述** | 自動取込ではカバーできない観察事項の追記 |
| **計画全体の整合性** | 取込項目が他の手書き項目と矛盾しないか |

### 特に注意が必要なケース

- **感覚トリガーが多い場合**: すべてが現在も有効か確認してください
- **医療フラグ**: 医療情報は必ず最新の状態と照合してください
- **環境調整**: 施設の実情に合わない環境調整が入っていないか確認してください

## 9. よくある質問

### Q: 取り込んだ内容を元に戻せますか？

A: 取込の Undo 機能はありません。
ただし、取込は「保存」前であればフォームの値を手動で編集・削除できます。
保存前に「キャンセル」を押せば、取込前の状態に戻ります。

### Q: 誰が取り込んだか後から分かりますか？

A: はい。取込履歴パネルに実行者名と日時が記録されます。
この情報はブラウザを閉じても保持されます。

### Q: 複数回取り込むとどうなりますか？

A: 同じデータは重複しません。
異なるアセスメント結果や、更新されたデータであれば追記されます。
取込履歴にはすべての取込操作が記録されます。

### Q: 別の職員が取り込んだ内容を見ることはできますか？

A: 同じブラウザ（同じ端末）であれば、取込履歴から確認できます。
異なる端末の取込履歴は、現時点では共有されません（localStorage ベース）。

### Q: 手順書の `[📋 計画]` バッジを非表示にできますか？

A: 現時点ではバッジの非表示機能はありません。
バッジは表示のみで、手順の編集や削除には影響しません。
