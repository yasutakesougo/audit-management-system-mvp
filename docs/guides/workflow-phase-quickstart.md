# 支援計画管理 — 1枚ガイド

> **「上から順に押すだけ」で PDCA が回ります。**

---

## Today ページで確認する

1. **今日の業務** 画面を開く
2. **📋 支援計画管理** カードを見る
3. 上にいる利用者ほど **優先度が高い**
4. 右側の **CTA ボタン** を押す → 該当画面へ移動

### 表示の見方

| マーク | 意味 | やること |
|:-------|:-----|:---------|
| 🔴 超過 | モニタリング期限を過ぎている | 今すぐモニタリングを実施 |
| 🟡 14日以内 | もうすぐモニタリング期限 | モニタリングの準備 |
| 🟡 再評価待ち | 再評価結果が未反映 | 計画を更新 |
| 🔵 未作成 | 計画シートがまだない | 計画シートを新規作成 |
| 🔵 設計中 | 手順が未設計 | 支援設計を続ける |
| 🟢 安定 | 問題なし | 次の期限まで運用継続 |

## 計画シート画面で進める

各タブを開くと、画面上部に **次のアクションバナー** が出ます。

### バナーの指示に従う

| タブ | よくある表示 | やること |
|:-----|:-----------|:---------|
| 概要 | 「支援手順が未設計です」 | `支援設計を続ける →` を押す |
| 概要 | 「モニタリング期限を過ぎています」 | `モニタリングを実施 →` を押す |
| モニタリング | 「見直し候補が検出されました」 | `再評価に反映 →` を押す |
| モニタリング | 「大きな見直し候補はありません」 | `再評価を確認 →` を押す |
| 再評価 | 「モニタリング結果を反映できます」 | `反映内容を確認 →` を押す |
| 再評価 | 「計画を更新してください」 | `計画を更新 →` を押す |

## PDCA の流れ

```text
① 計画シート作成（Plan）
     ↓
② 日次記録で支援実施（Do）
     ↓
③ モニタリングで振り返り（Check）
     ↓
④ 再評価で計画を見直し（Act）
     ↓
  ①に戻る
```

**このシステムでは、①〜④の「今どこにいるか」と「次に何をするか」が自動で表示されます。**

## よくある質問

### Q. バナーが出ない場合は？
**A.** 安定状態（🟢）です。次のモニタリング期限まで通常運用を続けてください。

### Q. 「超過」と「14日以内」の違いは？
**A.** 超過は期限を過ぎた状態です。速やかにモニタリングを実施してください。
14日以内は、まもなく期限が来る事前の通知です。

### Q. 誰がこの画面を見られますか？
**A.** サービス管理責任者（admin）のみです。直接支援員には表示されません。
