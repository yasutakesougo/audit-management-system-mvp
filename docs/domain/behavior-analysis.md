# 行動分析対象（支援手順記録対象）の定義と実装

> [!IMPORTANT]
> **SSOT (Single Source of Truth)**: 利用者フィールドの正定義および型情報は `src/features/users/schema.ts` を参照してください。

本ドキュメントでは、システム内における「行動分析対象（支援手順記録対象）」の定義および、コードから確認された実装上の事実を整理する。

## 1. コード上で確認された事実 (Implementation Facts)

### 判定ロジックとフラグ
利用者マスタ（SharePointリスト: `Users_Master`）上の以下の論理値フラグにより対象者を識別している。

- **`IsSupportProcedureTarget`**: 「支援手順記録対象」を判定する主フラグ。分析ダッシュボードの抽出等で使用。
- **`IsHighIntensitySupportTarget`**: 「強度行動障害支援対象」を示すフラグ。UI上の優先表示（ソート順）やバッジ表示で使用。
- **`severeFlag`**: スキーマ上に存在する既存フラグ。現在の実装では `IsHighIntensitySupportTarget` と役割が近接しており、併存している。

### 実装上の主な挙動
1. **優先表示 (Priority Sorting)**  
   `UserSelectionGrid` コンポーネントにおいて、`IsHighIntensitySupportTarget` が `true` の利用者をリストの先頭にソートするロジックが組み込まれている。

2. **機能の有効化・制限**  
   - `AnalysisDashboardPage` において、`IsSupportProcedureTarget` が `true` の利用者のみを選択肢として表示し、行動グラフやKPIを算出する。  
   - `IndividualSupportView` において、対象者支援に関連するタブ群（例：「支援計画書」「日々の記録」「PDCAサイクル」）を扱うUIが構成されている。

3. **アラート生成**  
   `useProactiveSPSAlerts` において、支援計画シート（SPS）の次回見直し期限（90日サイクル）や観察記録回数条件など、システム内で定義された条件に基づいてアラートを自動生成する。
