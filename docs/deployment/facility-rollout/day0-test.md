## ルール（必ず読んでから開始）

| ルール | 理由 |
|--------|------|
| ① 実際の利用者名・職員名を使う | 架空のIDでは「本気の入力」が再現できない |
| ② 「迷い」も記録する | エラーだけでなく「どこを押すかわからない」が改善のヒント |
| ③ 2人1組で実施 | 操作者は操作に集中。記録者が横で観察・記入 |
| ④ スマホ/タブレットでも最低1回試す | PCだけで「使える」としない |
| ⑤ 途中で止まったら番号をメモして次へ進む | 全てが通らなくても全体像を把握することが先 |

## 🚦 Go / Wait / Block 判定

テスト結果に基づいて、以下から1つに丸をつけてください。

### 🟢 Go（本番運用開始）
```
条件: S-1〜S-3 が全OK + S-4/S-5 に致命的なNGがない
→ 現場への展開を開始
```

### 🟡 Wait（条件付き延期）
```
条件: 1〜2個のUIの迷い（操作自体は完了している）
→ 改善点を記録し、修正後に再テスト
```

### 🔴 Block（導入停止）
```
条件: ログインできない / データが保存できない / 白画面になる
→ 原因を特定し修正するまで先に進まない
```
