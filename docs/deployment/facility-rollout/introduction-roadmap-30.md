## 📌 このロードマップの目的

**「一気に変えない。段階的に、確実に。」**

30名施設の日常を止めずに、紙の記録からシステムに移行します。
全4フェーズに分けて、**現場が慣れた段階で次に進む**設計です。

```
           Week 1-2          Week 3-4          Week 5-7          Week 8-10
        ┌───────────┐   ┌───────────┐   ┌───────────┐   ┌───────────┐
Phase 1 │ 準備と検証 │   │           │   │           │   │           │
        │ Day0テスト │   │           │   │           │   │           │
        └───────────┘   │           │   │           │   │           │
                    ┌───────────┐   │           │   │           │
            Phase 2 │ 主導線導入 │   │           │   │           │
                    │ 出欠+記録 │   │           │   │           │
                    └───────────┘   │           │   │           │
                                ┌───────────┐   │           │
                        Phase 3 │ 帳票+管理 │   │           │
                                │ 印刷+職員 │   │           │
                                └───────────┘   │           │

                                            │ 分析+計画 │
                                            └───────────┘
```

## Phase 1：準備と検証（Week 1〜2）

> **テーマ:** 「まず動くことを確認する」
> **対象者:** 管理者 + IT担当（2〜3名のみ）
> **現場への影響:** なし（裏側で検証するだけ）

### やること

| # | 項目 | 担当 | 完了条件 |
|---|------|------|---------|
| 1 | Microsoft 365 アカウント確認 | IT担当 | 全職員がログインできる |
| 2 | SharePoint リスト作成（12リスト） | IT担当 | `checkAllLists()` で 12/12 OK |
| 3 | 環境変数の設定 | IT担当 | アプリURLでログイン→ダッシュボード表示 |
| 4 | **Day0テスト実施** | 管理者2名 | [テストシート](day0_test_ready.md) で Go 判定 |
| 5 | テストデータの入力 | 管理者 | 利用者5名・職員3名の基本情報を登録 |

### Phase 1 → Phase 2 の Go 条件

```
✅ Day0テスト: Go 判定
✅ 利用者マスタ: 30名全員の基本情報が登録済み
✅ 職員マスタ: 全職員の情報が登録済み
✅ 引き継ぎ: テスト書き込み→別アカウントから閲覧 成功
```

### ⚠️ リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| ログインできない | 全停止 | IT担当と事前にMSAL設定を検証 |
| リスト名の不一致 | 画面が白くなる | `OpeningVerificationPage` で自動チェック |
| 利用者データの入力ミス | 記録が紐づかない | CSV一括インポートを使用 |

### 🗣️ 現場への伝え方

> 「今の業務はまったく変わりません。裏側でシステムの初期設定をしています。
> 皆さんに触っていただくのはPhase 2からです。」

## Phase 2：主導線の導入（Week 3〜4）

> **テーマ:** 「まず3つの画面だけ覚える」
> **対象者:** 全スタッフ
> **現場への影響:** 紙と並行運用（紙はまだ捨てない）

### 導入する機能（3つだけ）

| 機能 | 使う場面 | 紙からの変化 |
|------|---------|-------------|
| 🟢 **出欠記録** | 毎朝の登所時 | 出席簿 → タブレット入力 |
| 🟢 **日次支援記録** | 日中の支援後 | 手書きメモ → システム入力 |
| 🟢 **引き継ぎタイムライン** | 夕方の退勤前 | 口頭申し送り → システムに記載 |

### 2週間のスケジュール

| 日 | 内容 | 所要時間 |
|----|------|---------|
| Day 1（月曜） | **全体説明会** — 3画面の使い方デモ | 30分 |
| Day 1〜3 | 紙と**並行入力** — 紙が正、システムは練習 | 通常業務中 |
| Day 4（木曜） | **振り返りミーティング** — 困りごとの収集 | 15分 |
| Day 5〜7 | 紙と並行入力（困りごとを修正反映） | 通常業務中 |
| Day 8（翌月曜） | システム入力を**正本**に切り替え | — |
| Day 8〜14 | システムが正。紙はバックアップとして残す | 通常業務中 |

### Phase 2 → Phase 3 の Go 条件

```
✅ 全スタッフが出欠入力を1人でできる
✅ 支援記録が毎日入力されている（入力率 80%以上）
✅ 引き継ぎが口頭ではなくシステムで行われている
✅ 「白画面」「データ消失」の報告がゼロ
```

### ⚠️ リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| スタッフがログインできない | 記録できず紙に戻る | ログイン手順を印刷して掲示 |
| 入力が面倒で嫌がられる | 定着しない | 最初の2週間は紙を残し、心理的安全を確保 |
| タブレットが足りない | 入力の渋滞 | 共有タブレット2台＋個人スマホで分散 |
| Wi-Fiが不安定 | 保存エラー | オフライン対策を検討。保存失敗時は紙でバックアップ |

### 🗣️ 現場への伝え方

> 「今回お願いしたいのは、**たった3つの画面**だけです。
> まず2週間は紙と両方に書いてください。紙を捨てるのはまだ先です。
> 困ったら遠慮なく言ってください。すぐ対応します。」

## Phase 3：帳票・管理機能の導入（Week 5〜7）

> **テーマ:** 「月末の作業をラクにする」
> **対象者:** 管理者・サビ管（reception以上）
> **現場スタッフへの影響:** ほぼなし

### 導入する機能

| 機能 | 使う場面 | メリット |
|------|---------|---------|
| 🟢 **月次記録** | 月末の集計 | 手作業の集計が→自動化 |
| 🟢 **業務日誌プレビュー** | 監査用帳票の作成 | Excelコピペが→ワンクリック印刷 |
| 🟢 **職員出勤管理** | 勤怠管理 | 紙の出勤簿が→デジタル |
| 🟢 **スケジュール週間表** | 週間予定の共有 | ホワイトボードが→画面共有 |

### Phase 3 → Phase 4 の Go 条件

```
✅ 月次帳票で監査提出に使える品質の出力ができる
✅ 業務日誌の印刷が現行Excelと同等のレイアウトである
✅ Phase 2 の利用が安定している（入力率 90%以上に到達）
```

### 🗣️ 現場への伝え方

> 「スタッフの皆さんの作業は変わりません。
> 今回は**管理者側**の月末作業を楽にするための機能を追加します。
> 皆さんが毎日入力してくれたデータが、自動で月報になります。」

## Phase 4：拡張機能の選択的導入（Week 8〜10）

> **テーマ:** 「うちの施設に合うものだけ選ぶ」
> **対象者:** 管理者判断で個別に
> **注意:** 全施設一律ではなく、施設の特性に合わせる

### 選択可能な機能

| 機能 | こんな施設に向いている | 導入条件 |
|------|---------------------|---------|
| 🟡 **個別支援計画ガイド** | 計画更新が年2回以上ある | 次回計画更新時 |
| 🟡 **ISP比較エディタ** | 前回計画との差分を見たい | 管理者デモ後 |
| 🟡 **分析ダッシュボード** | データで傾向を把握したい | 1ヶ月のデータ蓄積後 |
| 🟡 **氷山モデルPDCA** | 行動支援の分析ニーズがある | 対象ケースがある場合 |
| 🟡 **コンプライアンスフォーム** | 自主点検チェックリストが必要 | 次回監査の1ヶ月前 |
| 🟡 **TodayOps ホーム** | 朝の「今日やること」を一画面で見たい | 観察期間後 |

### 🗣️ 現場への伝え方

> 「基本機能は十分使えるようになりました。
> ここからは**うちの施設に合うもの**を選んで追加していきます。
> 全部入れる必要はありません。必要なものだけ、管理者が判断します。」

## 🤝 サポート体制

## ❓ よくある不安と回答

| 不安 | 回答 |
|------|------|
| 「全部一気に変わるの？」 | いいえ。最初は3画面だけ。紙もまだ使います。 |
| 「パソコンが苦手でも大丈夫？」 | スマホで写真が撮れる人なら大丈夫です。操作は3ステップ以内です。 |
| 「データが消えたらどうする？」 | SharePoint（Microsoft 365）に保存されるので、誤削除しても復元できます。 |
| 「インターネットが切れたら？」 | その時は紙で記録→戻ったら入力。紙は最初の1ヶ月はバックアップとして残します。 |
| 「他の施設ではどうなの？」 | 同じ仕組みで運用中の施設があります。Phase 1〜2を重点的にサポートします。 |
| 「監査に通る？」 | A4横向きの業務日誌が出力できます。現行Excelと同等のレイアウトです。 |
| 「費用は？」 | 既存のMicrosoft 365ライセンス内で動きます。追加費用はありません。 |
