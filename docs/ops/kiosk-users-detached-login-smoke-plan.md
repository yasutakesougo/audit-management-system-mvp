# `/kiosk/users` 別事業所向けログイン確認手順（smoke plan）

## 目的

別事業所職員1名で、`/kiosk/users` への到達までが実環境で可能かを確認する。

保存・更新・削除は行わず、認証・権限・遷移可否のみ確認する。

## 対象

- 対象者: 別事業所職員1名（法人許可済み）
- 環境: 試行先の別環境URL（運用前提）

## 確認URL

- 基本: `/kiosk/users`
- 必要に応じて `?kiosk=1` を付与してもよい

## 確認しない操作

- 利用者情報の新規作成
- 支援手順の新規作成・更新
- 実施記録の保存/更新/削除
- SharePointデータの手動編集

## チェック項目

- [ ] Microsoft 365 サインインできる
- [ ] MFAで即時停止しない（`Challenge` が発生しても正常に完了できる）
- [ ] アプリ側で認証拒否されない
- [ ] `/kiosk/users` が表示される
- [ ] SharePoint権限制限が画面に表示されず、読み取り導線が成立する

## 中止条件（このフェーズ）

- サインイン時点で対象アカウントが拒否される
- MFA で進行不能
- `/kiosk/users` に到達できない
- 保存なしでも権限不足で業務導線が成立しない

## 補足

- ログイン確認で停止した場合は、保存・更新・削除を実行しない。
- この手順完了前に実装・deploy・SharePoint作成・env/secret変更を進めない。
