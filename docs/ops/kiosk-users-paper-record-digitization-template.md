# `/kiosk/users` 紙記録データ化テンプレート

## 1. 目的

このテンプレートは、紙記録で運用している別事業所が `/kiosk/users` 支援手順記録を試行する前に、利用者情報と支援手順を安全にデータ化するための記入基準である。

このPRでは、実装、テスト追加、SharePoint作成、デプロイ、環境変数変更、シークレット変更、本番データ変更は行わない。

試行記録は、原則として正式記録ではなく、紙記録または既存運用の補助記録として扱う。正式記録化する場合は、監査、保存年限、訂正履歴、承認責任、紙記録廃止条件を別途決める。

## 2. 利用者マスタ入力項目

| 項目 | 記入内容 | 必須 | 記入基準 |
| --- | --- | --- | --- |
| 利用者ID | 試行環境内で一意なID | 必須 | 既存本番のIDと衝突しない値にする |
| 利用者表示名 | キオスクで表示する名前 | 必須 | 現場職員が取り違えない表記にする |
| 所属事業所 | 試行対象の事業所名 | 必須 | 既存事業所名と混同しない表記にする |
| 事業所コード | `OrgCode` / `facilityId` 相当 | 必須 | 将来統合時に識別できる値にする |
| 有効状態 | 有効 / 無効 | 必須 | 試行対象外は無効にする |
| 支援手順記録対象 | 対象 / 対象外 | 必須 | `/kiosk/users` に表示してよい利用者だけ対象にする |
| 紙記録の保管場所 | 原紙または写しの所在 | 必須 | 後から突合できる単位で記録する |
| 主記録の扱い | 紙記録 / デジタル補助 / 正式記録候補 | 必須 | 初期試行では原則「紙記録」または「デジタル補助」 |
| 備考 | 試行前に共有すべき補足 | 任意 | 医療・安全上の詳細は必要最小限にする |

## 3. 支援手順マスタ入力項目

| 項目 | 記入内容 | 必須 | 記入基準 |
| --- | --- | --- | --- |
| 手順ID | 試行環境内で一意なID | 必須 | 利用者ID、日付、行番号だけに依存しない |
| 利用者ID | 対象利用者のID | 必須 | 利用者マスタと一致させる |
| 手順名 | 現場で識別できる短い名前 | 必須 | 画面上で読める長さにする |
| 実施タイミング | いつ実施するか | 必須 | 時刻、活動名、場面名のいずれかで明確にする |
| 本人の動き | 本人に想定される行動 | 必須 | 観察可能な表現にする |
| 支援者の手順 | 職員が行う支援 | 必須 | 具体的な声かけ、介助、環境調整を書く |
| 注意事項 | 実施時に気を付けること | 必須 | 誤解すると事故につながる内容を優先する |
| 禁忌事項 | やってはいけない対応 | 必須 | 「禁止」だけでなく理由または代替対応も書く |
| 更新日 | 紙記録または確認日の更新日 | 必須 | 古い手順をそのまま試行対象にしない |
| 作成者 | 紙記録またはデータ入力の作成者 | 必須 | 入力者とは別に紙記録上の作成者がいる場合は分ける |
| 確認者 | 内容を確認した現場責任者 | 必須 | 支援内容を理解している人に限定する |
| 承認者 | 試行対象にしてよいと判断した人 | 必須 | 管理者またはサービス管理責任者を想定する |
| 紙記録参照 | 原紙、ページ、行、ファイル名など | 必須 | 後から原紙へ戻れる粒度で記録する |

## 4. 禁忌・注意事項の書き方

禁忌・注意事項は、現場職員がその場で判断できる表現にする。

| 区分 | 書いてよい例 | 避ける例 |
| --- | --- | --- |
| 禁忌 | 大きな声で急かさない。拒否が強い場合は5分空けて再提示する。 | 怒らせない |
| 禁忌 | 身体接触が必要な場合は、先に声をかけて反応を確認する。 | 無理にしない |
| 注意事項 | 午前は疲労が出やすいため、移動前に表情と歩行状態を確認する。 | 様子を見る |
| 注意事項 | 水分摂取を促す場合は、選択肢を2つまでにする。 | 声かけする |

「なぜ危険か」「代わりに何をするか」が分からない表現は、確認者に差し戻す。

## 5. 記録に残してよい表現

記録に残す表現は、事実、観察、実施した支援、本人の反応を分ける。

| 区分 | 書いてよい例 | 避ける例 |
| --- | --- | --- |
| 事実 | 10:15 に作業室から休憩スペースへ移動した。 | いつものように移動した。 |
| 観察 | 声かけ後、約30秒間返答がなかった。 | やる気がなかった。 |
| 支援 | 選択肢を2つ提示し、本人が選んだ方を実施した。 | うまく誘導した。 |
| 反応 | 首を横に振り、着席を継続した。 | 嫌がった。 |

人格評価、推測、責任追及に見える表現は避ける。必要な場合は、具体的な行動や観察事実へ置き換える。

## 6. 入力者 / 確認者 / 承認者

紙記録からのデータ化では、入力、確認、承認を分ける。

| 役割 | 責任 | 完了条件 |
| --- | --- | --- |
| 入力者 | 紙記録をテンプレートへ転記する | 必須項目が埋まり、紙記録参照が残っている |
| 確認者 | 支援内容が現場実態と合っているか確認する | 古い手順、曖昧表現、禁忌漏れを確認済み |
| 承認者 | 試行対象として使ってよいか判断する | 試行対象、対象外、保留のいずれかを決める |

入力完了だけでは試行対象にしない。確認者または承認者が未記入の場合は、試行開始不可とする。

## 7. 紙記録との対応関係

データ化した項目は、必ず紙記録へ戻れるようにする。

| 項目 | 記入欄 |
| --- | --- |
| 紙記録名 |  |
| 原紙 / 写し / PDF / 写真 |  |
| 保管場所 |  |
| 対象ページ |  |
| 対象行または見出し |  |
| 紙記録の更新日 |  |
| データ化した日 |  |
| 入力者 |  |
| 確認者 |  |
| 承認者 |  |
| 差し戻し理由 |  |

紙記録との対応関係が不明な手順は、試行対象にしない。

## 8. 試行対象にできる条件

次をすべて満たす場合のみ、試行対象にできる。

- [ ] 利用者ID、利用者表示名、所属事業所が埋まっている。
- [ ] 支援手順の実施タイミングが明確である。
- [ ] 支援者の手順が具体的な行動として書かれている。
- [ ] 注意事項と禁忌事項が確認済みである。
- [ ] 紙記録との対応関係が残っている。
- [ ] 入力者、確認者、承認者が記録されている。
- [ ] 主記録が紙記録かデジタル補助か明記されている。
- [ ] 既存本番データ、既存SharePoint、既存認証とは共有しない前提で整理されている。
- [ ] 障害時に紙記録へ戻れる。

## 9. 試行対象にしない条件

次のいずれかに該当する場合は、試行対象にしない。

- [ ] 紙記録の所在または更新日が不明である。
- [ ] 支援手順の作成者、確認者、承認者が不明である。
- [ ] 禁忌事項が空欄、または「特になし」の根拠が不明である。
- [ ] 手順が古く、現場実態と合っているか確認できない。
- [ ] 記録表現に人格評価、推測、責任追及が残っている。
- [ ] 既存本番の利用者ID、SharePoint、認証、デプロイ先と混同する可能性がある。
- [ ] 試行中止時に紙記録へ戻る手順が決まっていない。

対象外にした場合は、理由と再確認担当者を残す。
