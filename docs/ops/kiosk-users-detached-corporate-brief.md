# 法人側確認用ブリーフ（#2381 参照）

別事業所で `/kiosk/users` の複製試行を始める前提として、既存本番と分離した環境運用の方針を整理しました。

今回は実装着手ではなく、試行準備に進む前提条件の確認です。

- 別URLを作成してよいか
- 別事業所向けのSharePointを作成してよいか（サイト or 別リスト群）
- secret管理者を誰にするか
- 別事業所職員1名でログイン確認してよいか
- 紙記録を正式記録として継続し、アプリ記録は補助記録として扱ってよいか

これらが合意されるまでは、以下は行いません。

- deploy
- SharePoint作成/変更
- env/secret変更
- 本番データ変更
- 保存・更新・削除を伴うアプリ操作
