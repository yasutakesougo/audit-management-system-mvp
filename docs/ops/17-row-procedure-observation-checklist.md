# 17行標準支援手順 継続観測チェックリスト

## 1. 観測の目的
17行標準支援手順モデル（`PROCEDURE_ROWS` / `dailyProcedureMapper`）が、本番環境の多様な実データにおいて意図通りに機能しているかを継続的に検証する。E2Eテストでカバーしきれない現場データの揺れや、表示の違和感を早期に発見し、信頼性を高めることを目的とする。

## 2. 対象利用者の選定基準
以下の異なるデータパターンを持つ利用者を計3〜5名選定する。

- [ ] **構造化手順あり**: `procedureSteps` に行番号（1〜17）が指定されている利用者
- [ ] **構造化手順なし**: 構造化ステップがなく、基本ステップ（17行）のみが表示される利用者
- [ ] **外活動あり**: 16行目（外活動準備）、17行目（外活動）に手順が設定されている利用者
- [ ] **旧データ/レガシーテキストあり**: 構造化ステップがなく、「対応方針」「具体策」「環境調整」にテキスト入力がある利用者（Aggregationの検証）
- [ ] **入力密度が低い**: 未設定項目が多く、ほとんどの行が空欄またはデフォルト表示の利用者

## 3. 観測対象日の選定
各利用者について、以下の状況の異なる日を確認する。

- [ ] **今日**: 現在の運用状態
- [ ] **前日以前**: 既に記録が作成・保存されている日（再読込の安定性検証）
- [ ] **未来日**: 計画データのみが反映され、記録が未作成の状態
- [ ] **記録未作成の日**: 初回表示時のデフォルト展開の検証

## 4. チェック項目
ProcedurePanelを開き、以下の観点で表示を確認する。

### 基本構造
- [ ] **17行の表示**: 全ての行（1〜17）が表示されているか
- [ ] **行順序（rowNo）**: 1〜15の時系列順および16〜17の外活動が正しい順序で並んでいるか
- [ ] **data-row-no の存在**: 各行のDOM要素に `data-row-no="1"` 〜 `"17"` が付与されているか
- [ ] **空欄時の表示**: データがない行でもレイアウトが崩れず、プレースホルダーやデフォルト活動名が表示されているか

### コンテンツ反映
- [ ] **本人の動き**: 支援計画の「本人の動き」が正しく表示されているか
- [ ] **支援者の動き**: 支援計画の「支援者の支援」が正しく表示されているか
- [ ] **留意事項（condition）**: 「環境調整」や「様子・条件」が正しく表示されているか
- [ ] **外活動（16/17行）**: 外活動の設定が日中活動行から切り離されて16/17行に表示されているか

### 特殊ロジック
- [ ] **Legacy Text Aggregation**: 構造化手順がない場合、対応方針等がAM/PM活動行に自然に集約されているか
- [ ] **ID/状態の安定性**: ウィザード内での「戻る/次へ」移動や、保存・再読込を繰り返しても行番号や内容がズレないか

## 6. 判定基準（WARN / FAIL の分類）

### WARN（警告）: 運用は可能だが改善が望ましい
- 表示上の微細な揺れ（マージンやフォントサイズの違和感）
- 文言の軽微なニュアンスの違和感
- Legacy Text Aggregation の結果、改行位置が不自然など

### FAIL（失敗）: 修正が必要な不具合
- 行の欠落、重複、順序の入れ替わり
- データの消失（計画にあるのに表示されない、保存後に消える）
- `data-row-no` の欠落や重複（E2Eテストが壊れる原因）
- 構造化手順があるのに Legacy Aggregation が重複して発動する等

## 7. 問題発見時の対応方針
観測フェーズでは**「発見と記録」**に徹し、その場でのコード修正は原則行わない。

1. **再現条件の特定**: 利用者ID、対象日、具体的な操作手順を記録する。
2. **Issue化**: `rowNo` と不具合内容を特定し、構造的な問題かデータ依存の問題かを切り分ける。
3. **PR分離**: 修正が必要な場合は、影響範囲を最小限にした「小さな修正PR」として別途作成する。

## 8. 推奨確認コマンド

開発環境の起動:
```bash
npm run dev
```

E2Eテストによる自動検証:
```bash
# 全体の検証
npx playwright test tests/e2e/procedure-17row-bridge.spec.ts --project=chromium

# 特定の利用者での挙動を確認する場合（デバッグ用）
npx playwright test tests/e2e/procedure-17row-bridge.spec.ts --project=chromium --debug
```

ユニットテスト（マッピングロジック単体）:
```bash
npm test src/features/planning-sheet/logic/dailyProcedureMapper.spec.ts
```
