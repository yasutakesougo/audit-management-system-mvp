# コーヒー精算 ミニ運用手順

## 対象

コーヒー注文の月次確認、精算状態の更新、現金・紙運用との突合を行うスタッフ。

## 開き方

- キオスク端末では下部ナビの「コーヒー精算」を押す。
- 直接開く場合は `/billing` を開く。
- 表示が古い、または精算状態が反映されない場合はブラウザを再読み込みする。

## 月末の確認担当

| 役割 | 担当 | 作業 |
| --- | --- | --- |
| 一次確認 | 現場担当 | 対象月の注文数・金額・未精算者を確認する |
| 精算更新 | 精算担当 | 入金確認後に「精算済み」へ変更する |
| 最終確認 | 管理担当 | CSVまたは印刷結果を保管し、紙・現金運用との差分を確認する |

## 精算済みにするタイミング

1. 対象月を選ぶ。
2. 未精算者の注文数・金額を確認する。
3. 現金または紙記録で入金済みを確認する。
4. 対象者の精算状態を「精算済み」に変更する。
5. 画面を再読み込みし、状態が戻らないことを確認する。

未入金、金額不一致、対象者不明の場合は「未精算」のままにする。

## 間違えた記録を見つけた時

- 注文数が違う: 注文履歴側の記録日・提供済み状態を確認する。
- 対象者が違う: 氏名だけで判断せず、利用者コードまたは注文者コードを確認する。
- 精算済みにしすぎた: 同じ行を「未精算」に戻し、再読み込み後も戻っているか確認する。
- 判断できない: その場で修正せず、問題記録テンプレートに残して管理担当へ共有する。

## 赤帯・エラー・警告が出た時

以下を記録して、管理担当または開発担当へ共有する。

```text
発生日:
画面:
対象月:
利用者:
表示された文言:
操作内容:
再読み込みで直ったか:
スクリーンショット:
```

特に「精算状態の永続化列が未設定です」という警告が出た場合、SharePoint の `PaymentStatus`, `PaidAt`, `PaidBy` 列確認が必要。

## 紙・現金運用との併用期間

- 受け入れ確認中の1週間は、紙・現金表も従来どおり残す。
- 毎日、画面の未精算額と紙・現金表を突合する。
- 1週間連続で不一致がなければ、紙側は控え用途へ縮小できる。
- 不一致が出た場合は、画面だけを正とせず、注文履歴・現金表・精算状態の3点を確認する。

## 月末作業の最小チェック

| 項目 | 結果 |
| --- | --- |
| 対象月が正しい | [ ] |
| 未精算者が確認済み | [ ] |
| 精算済み変更後に再読み込みした | [ ] |
| CSVを出力した | [ ] |
| 印刷またはPDF保管した | [ ] |
| 紙・現金表との差分がない | [ ] |
