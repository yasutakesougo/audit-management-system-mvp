## 4. 項目別棚卸し（実施前チェック）

## 4.1 別URL / deploy方式
- 候補A: 別ドメイン（独立のベースURL）
- 候補B: 同一ドメインで別パス（`/pilot-*`）
- 要判定:
  - 職員が誤遷移しない命名規則
  - 既存本番のブックマーク/QR/端末設定からの隔離可否
  - 監査ログ・運用連絡窓口での識別性
- 状態: `HOLD`  
- 推奨: 別URL採用（別ドメインを最優先）
- 決定者: 法人システム管理責任者
- エビデンス: `docs/ops/kiosk-users-detached-environment-runbook.md`

## 4.2 別SharePoint（サイト or リスト群）
- 候補: 別サイト、別リスト群（現行サイト＋分離リスト）  
- 要判定:
  - 既存本番リスト/列/権限への書き込み防止
  - Power Automate・監査ログ配信先の分離
  - 利用者・手順・実施記録の混入防止
- 状態: `HOLD`  
- 推奨: 別サイト方式（初期の分離境界が明示しやすい）
- 決定者: SharePoint運用責任者
- エビデンス: `docs/ops/kiosk-users-detached-environment-runbook.md`

## 4.3 認証対象職員（別事業所）
- 要判定:
  - 1次試験職員（少なくとも1名）を本人確認済みアカウントで確保
  - 既存本番対象外職員が入れないことを確認
  - 退職・異動時の即時停止運用
- 状態: `HOLD`
- 決定者: 利用者事業所管理者 + 現場監修者
- エビデンス: 試験対象者一覧（作成待ち）

## 4.4 Microsoft 365サインイン条件
- 確認項目:
  - MFA要否と例外可否
  - 条件付きアクセス例外（ネットワーク / 端末条件）
  - サインインフロー（イントラ/外部）と期限系ポリシー
- 状態: `HOLD`
- 決定者: M365管理者
- 次アクション: 1名のログイン試験まで実測し、拒否時の原因を切り分け

## 4.5 redirect URL要否
- 要判定:
  - 認証アプリ登録側のリダイレクト許可リスト
  - 本番/試行で別URLが生じる場合の追加URL要否
  - ローカル環境・ブラウザ別の例外条件
- 状態: `HOLD`
- 次アクション: アプリ構成レビュー依頼（実URL確定後）

## 4.6 環境変数キー分離
- 事前整理対象（暫定）:
  - デプロイ先識別キー
  - API基底URL/エンドポイント
  - テナント識別子、facility識別キー
  - 認証設定（clientID/issuer/audience等）
  - ログ/監視識別キー
- 状態: `HOLD`
- 方針: 既存本番キーと同名キーは原則 `suffix/prefix` で分離し、読み取り先を明示

## 4.7 secret管理者
- 要決定:
  - 運用責任者（1名）
  - 更新承認者（1名）
  - 秘密情報保有者（2名以上の分離推奨）
- 状態: `BLOCK`（現時点で未確定）
- 次アクション: 法人側の名簿で管理者2名を確定し、権限差し替え禁止ルールを固定

## 4.8 利用者マスタ初期データ
- 必須項目（`kiosk-users-paper-record-digitization-template.md` 準拠）:
  - 利用者表示名 / 利用者ID / 所属事業所 / 有効無効 / 紙記録リンク
  - 更新日 / 入力者 / 確認者 / 承認者
  - 紙記録との対応
- 対象: 試行対象からまず3〜5名
- 状態: `HOLD`
- 条件: 入力・確認・承認者の分離を確認済みでない限り試行対象不可

## 4.9 支援手順マスタ初期データ
- 必須項目:
  - 利用者ID / 手順名 / 実施タイミング / 注意・禁忌 / 更新日
  - 作成者 / 確認者 / 承認者
  - 紙記録参照（ページ・行・日時）
- 対象: 利用者3〜5名分の最小データ
- 状態: `HOLD`
- 条件: 人格評価・推測表現を残さず、観察事実ベースで修正済み

## 4.10 紙記録併用ルール
- 原則: 紙記録を主記録として継続
- 試行記録は補助扱い
- 不一致時の解決: 紙記録を優先、相違理由・対応者・日付を残す
- 状態: `GO`
- 共有場所: 運用向け説明文 + runbookリンク

## 4.11 中止条件（試行前再確認）
- 本番・別事業所・対象外利用者データの混在
- MFAや認証条件で外部職員が誤入場
- 重要データの責任者未確定
- 紙記録へ戻せない運用状態
- 入力者・確認者・承認者の分離が崩れる
- SharePoint/認証/URL分離を説明できない
