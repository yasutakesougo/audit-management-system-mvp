# キオスク誤認防止 UX 点検

## 目的

キオスク画面で「読み込み失敗」「対象者なし」「未保存」「保存済み」が見た目で混同されないか確認する。新しい入力機能を増やす前に、現場スタッフが状態を誤認しないことを優先する。

## 対象画面

| 画面 | パス | 主な誤認リスク |
| --- | --- | --- |
| キオスクホーム | `/kiosk` | どこから戻るか分からない |
| 利用者選択 | `/kiosk/users` | 読み込み失敗を対象者なしと誤認する |
| 支援手順一覧 | `/kiosk/users/:userId/procedures` | 誰の記録か分からなくなる |
| 支援手順詳細 | `/kiosk/users/:userId/procedures/:slotKey` | 保存済み・未保存が分からない |
| トイレ確認 | `/kiosk/toilet` | 読み込み失敗と対象者なしが似る |
| コーヒー精算 | `/billing` | キオスクから出た後の戻り先が分からない |

## 点検観点

| ID | 観点 | 期待 | 結果 |
| --- | --- | --- | --- |
| K1 | 戻る導線 | すべての画面でホームまたは前画面へ戻れる | [ ] OK / [ ] NG |
| K2 | 対象者表示 | 記録対象の氏名・コードが常に確認できる | [ ] OK / [ ] NG |
| K3 | 保存状態 | 保存中・保存済み・未保存が別表示になっている | [ ] OK / [ ] NG |
| K4 | 読み込み失敗 | エラー文言・再試行導線があり、対象者なしと区別できる | [ ] OK / [ ] NG |
| K5 | 対象者なし | 正常に読み込めたが対象者がいない状態として表示される | [ ] OK / [ ] NG |
| K6 | 通信失敗 | 連打しにくい、または保存中ボタンが無効化される | [ ] OK / [ ] NG |
| K7 | 再読み込み | 保存済み状態が再読み込み後も確認できる | [ ] OK / [ ] NG |
| K8 | キオスク外画面 | `/billing` などへ遷移後もキオスクへ戻れる | [ ] OK / [ ] NG |

## 状態別の期待表示

| 状態 | 表示の期待 | NG例 |
| --- | --- | --- |
| 読み込み中 | ローディング、または「読み込み中」 | 空白だけ |
| 読み込み失敗 | エラー文言、再試行、原因のヒント | 「対象者がいません」だけ |
| 対象者なし | 正常取得後の空状態であることが分かる | エラーと同じ赤帯 |
| 保存中 | ボタン無効化、進行中表示 | 何度も押せる |
| 保存済み | 保存時刻、対象者、完了表示 | 完了したか分からない |
| 未保存 | 入力中・下書き・変更ありの表示 | 保存済みと同じ見た目 |


## 修正優先度

P1:
- 読み込み失敗が対象者なしに見える
- 保存失敗が保存済みに見える
- 対象者を取り違える可能性がある

P2:
- 戻る導線が分かりにくい
- 保存中に連打できる
- 再読み込み後の状態確認が分かりにくい

P3:
- 文言の統一
- 表示位置や余白の改善
- アイコンや色の補助
