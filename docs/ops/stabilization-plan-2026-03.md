# ⚓️ 7日間安定化フェーズ 運用マニュアル (2026-03-31 〜 04-06)

## 🎯 最優先指標 (KPI)
- **QuickRecord 保存成功率**: 99.9% 以上の維持。
- **支援記録入力完遂率**: 当日 18:00 時点での Pending ゼロ化。
- **指標確認**: 毎夕 17:00 に Nightly Patrol / Telemetry 経由で観測。
※ これら指標の悪化時のみ、緊急改善を実施する。

## 🛡 変更管理ルール (1KPI = 1PR)
- 安定化フェーズ中のコード変更は「1つの KPI 改善」に対し「1つの PR」のみとする。
- 複数の修正・リファクタリングを1つの PR に混ぜることを厳禁とする。
- 修正理由が KPI に直結しない「ついで」の変更はすべて Wave 3 へ回す。

## 🚨 緊急停止基準 (ROLLBACK判断)
- **Runtime Error**: ExceptionCenter にて同一原因エラーが **5分以内に 10件以上** 発生。
- **機能停止**: QuickRecord の保存が 3回連続で失敗、または画面が白転。
- **整合性欠如**: 現場から「保存したはずのデータが消えた」という報告が 1件でも入った場合。

## 📅 デイリールーチン
1. **09:00 Nightly Triage**: 昨夜の巡回結果から優先 Issue を抽出。
2. **13:00 Telemetry Check**: 午前中の利用状況と Exception 発生状況の確認。
3. **17:00 KPI Review**: 保存成功率および完遂率の最終チェック。

## 🏁 フェーズ終了条件 (2026-04-07)
- 7日間、重大なシステム停止（Severity 1）が発生していないこと。
- `any` 型や巨大ファイルの増加率が抑制されていること。
- 現場ユーザーからの「操作不能」報告が 0件であること。
