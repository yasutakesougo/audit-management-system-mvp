# 申し送り統合システム PoC 導入ガイド

**目的**: 申し送り・朝会・夕会・日次記録の業務効率化
**PoC期間**: 2週間（推奨）
**対象**: 常勤職員・非常勤職員・管理者

---

## 🎯 このシステムで何が楽になるか

### Before（現在の課題）
- ❌ 申し送りが口頭中心で記録が残らない
- ❌ 朝会・夕会で重要事項の見落としが発生
- ❌ 日次記録の特記事項を一から手入力
- ❌ 重要な情報が各スタッフの記憶に依存

### After（システム導入後）
- ✅ 申し送り事項がタイムライン形式で記録・共有
- ✅ 朝会・夕会で重要案件を自動アラート表示
- ✅ 重要な申し送りが日次記録の特記事項に自動転記
- ✅ カテゴリ別の傾向分析で管理者の状況把握が向上

## ⏰ 1日の標準フロー

### 🌅 朝会（08:50-09:10）

#### 司会者の進行（MeetingGuideページ使用）
1. **画面を開く**: MeetingGuideページ → 朝会モード
2. **申し送りチェック**: Drawer内のミニサマリーで昨日の重要事項確認
3. **アラート対応**: 重要案件がある場合は詳細を確認
4. **共有**: 「○○さんの体調面で注意」「△△の件は対応済み」等を口頭共有

#### 参加者
- Dashboard画面で個別に気になる事項をチェック
- 必要に応じて申し送りタイムライン詳細を確認

### 🌆 夕会（16:30-17:00）

#### 司会者の進行
1. **画面を開く**: MeetingGuideページ → 夕会モード
2. **今日のサマリー**: 今日の申し送り件数・カテゴリ確認
3. **未対応確認**: 「未対応」の申し送りを確認し、対応状況を口頭確認
4. **状態更新**: 対応済みの案件を「対応済み」に更新

#### 参加者
- 自分が気づいた事項があれば申し送りタイムラインに追加
- 明日への引き継ぎ事項があれば「重要」レベルで入力

### 📝 日次記録作成（17:00-17:30）

1. **フォーム表示**: 日次記録作成フォームを開く
2. **自動転記確認**: 特記事項に重要な申し送りが自動で下書きされているか確認
3. **内容調整**:
   - 不要な行は削除
   - 追加で書きたいことがあれば手動追記
4. **保存**: そのまま保存して完了

## 📱 画面別：使い方のポイント

### 申し送りタイムライン
**URL**: `/handoff-timeline`

#### 📝 書き方のコツ
- **短文で具体的に**: 「○○さん、血圧高め。要観察」
- **カテゴリ選択**: 体調・行動面・家族連絡等から適切に選択
- **重要度設定**:
  - `重要`: 次のスタッフが必ず知るべき事項
  - `要注意`: 気にかけてほしい程度
  - `通常`: 記録として残す程度

#### 🎯 状態管理
- **未対応**: 入力直後の初期状態
- **対応中**: 確認・対応を開始した状態
- **対応済み**: 完了・問題なしの状態

### 朝会・夕会ガイド
**URL**: `/meeting-guide`

#### 🎪 司会者向けのポイント
- Drawer内の「申し送りミニサマリー」を最初に確認
- アラートバッジ（赤・黄・緑）で優先度を判断
- 重要案件は必ず口頭で共有
- 対応済みの案件は状態を「対応済み」に更新

### 日次記録フォーム

#### 🔄 特記事項の自動連動について
- **自動転記**: 重要度「重要」の申し送りのみ自動で特記事項に下書き
- **編集可能**: 自動で入った文章は自由に削除・編集OK
- **表示形式**:
  ```
  【申し送り（重要）から自動転記】
  ・⚠️ 09:30 体調：血圧が高めで注意が必要
  ・🔄 14:15 行動面：他利用者との衝突があった
  ```
- **ルール**: 新規作成時のみ動作（編集時は自動転記なし）

## ❓ よくある質問

**Q: 申し送りの入力は必須ですか？**
A: 強制ではありません。気づいた時に気づいたことを入力する形で始めてください。

**Q: 既存の紙の申し送りはどうしますか？**
A: PoC期間中は並行運用でOK。徐々にシステム中心に移行していきましょう。

**Q: システムが使えない時間があったらどうしますか？**
A: 従来通り口頭・紙での申し送りをバックアップとして残しておいてください。

**Q: 日次記録の特記事項に不要な内容が自動で入ることはありますか？**
A: 重要度「重要」のもののみ転記されます。不要な行は自由に削除してください。
