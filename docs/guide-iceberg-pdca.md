# 氷山PDCAページの使い方

## 📌 このページの目的

利用者の行動の**背景（環境・感情・ニーズ）を氷山モデルで分析し、
改善を PDCAサイクルで継続的に管理する支援設計ツール**です。

日々の支援記録から得た洞察を構造的に整理し、
チームで共有しながら支援を改善していきます。

### 氷山モデルとは

```
      🏔️ 見える部分
    ──────────────
    行動（問題行動など）
    ══════════════
    ～～～水面～～～
    ══════════════
    感情
    ──────────────
    認知
    ──────────────
    環境要因
    ──────────────
    本人のニーズ
    ──────────────
      🧊 見えない部分
```

表面に見える「行動」の下に、感情・認知・環境・ニーズが隠れています。
このページは本質的には**「問題行動の原因分析ツール」**であり、
見えない部分を構造的に整理して、根本的な支援改善につなげます。

---

## 🔗 URL の意味

```
/analysis/iceberg-pdca?userId=14&source=monitoring
```

| パラメータ | 意味 |
|-----------|------|
| `userId=14` | 利用者ID 14（例：田中 太郎）に絞り込んで表示 |
| `source=monitoring` | モニタリング画面から遷移してきたことを追跡 |

---

## 🖥️ 画面の構成

### ① メトリクスエリア（上部）

- **当日入力完了率**: 今日の支援記録の入力完了率
- **未送信解消リードタイム**: 未送信のまま放置された時間の平均
- **週次 / 月次トレンド**: 直近7日 or 30日の推移を切り替えて確認
- **「対象日の支援記録へ」リンク**: 当日の支援記録ページへ直接移動

支援記録の入力状況や支援の改善傾向を確認できます。

### ② 利用者フィルター（中央）

利用者を切り替えてPDCA分析を行います。
URLパラメータ `userId` でプリセットされている場合、最初からその利用者が選択されます。

### ③ PDCA新規作成フォーム

| 入力項目 | 内容 |
|---------|------|
| **タイトル** (必須) | 課題テーマ（例：「午後外出時の不安行動」） |
| **概要** | 気づき・背景の詳細 |
| **Phase** | PLAN / DO / CHECK / ACT から選択 |

入力後「作成」ボタンで保存します。

### ④ PDCA一覧

作成したPDCAがカード形式で表示されます。

| 操作 | 説明 |
|------|------|
| **編集** | フォームに再読み込みして修正 |
| **削除** | 確認ダイアログ後に削除 |
| **支援計画に反映** | ACTフェーズのみ。支援計画のモニタリングへジャンプ |

---

## 💡 典型的な使い方の流れ

### ① モニタリング → 氷山PDCAへ遷移

モニタリング画面から「分析」ボタンで遷移（`source=monitoring`）

### ② PLAN 作成（計画）

> 例：「午後の外出活動時に不安が増し、自傷行為が見られる」
>
> 氷山モデル分析：
> - 行動：腕を叩く
> - 感情：不安・恐怖
> - 環境：騒がしい場所、予定の急な変更
> - ニーズ：見通しを持ちたい

### ③ DO（実行）

> 例：「事前にビジュアルスケジュールを見せて見通しを持てるようにした」

### ④ CHECK（評価）

> 例：「3日間の観察で不安行動が50%減少。特に予定表を確認した直後は落ち着いている」

### ⑤ ACT（改善）

> 例：「見通しシートを標準支援として採用。全場面で活用する」

### ⑥ 支援計画へ反映

ACTフェーズでは、有効だった支援方法を**支援計画シートに正式に反映**できます。

これにより：
- ✅ 支援の成功事例をチームで共有
- ✅ 次回モニタリング資料として利用
- ✅ 支援計画シートの改善履歴として保存

---

## 🔄 システム全体の流れ

```
日々の支援記録
      │
      ▼
氷山PDCA（気づきの整理・原因分析）
      │
      ▼
モニタリング会議（3ヶ月ごと）
      │
      ▼
支援計画シート更新
```

この流れは制度上の要件にも直結します：

| 制度要件 | 対応する機能 |
|---------|------------|
| アセスメント記録 | 支援記録 + 氷山PDCAのPLAN |
| 3ヶ月モニタリング | モニタリング会議 + CHECK/ACT |
| 会議議事録 | モニタリング記録 |
| 支援計画の見直し | ACT → 支援計画シートへの反映 |

## ⚙️ 技術的な補足

- データは SharePoint / localStorage に保存（環境による）
- `VITE_WRITE_ENABLED=1` で書き込み可能（それ以外は閲覧のみ）
- admin ロールは常に書き込み可能
- feature flag `icebergPdca` が有効な場合のみアクセス可能
