# Audit Management System MVP Kiosk Operational Manual / Field Workflow Readiness

調査実施日: 2026年6月19日
対象コミット: `7dde895340c5a6914f2c12ecd858621d5821a105`

## 1. 調査目的
本手順書は、施設内に設置された共有タブレット等の端末で使用される「キオスク（Kiosk）モード」の日常の操作方法と、現場での運用基準を定めるものです。
現場の支援職員が迷わず正確に日中活動や排泄などの支援実績を記録でき、ネットワークの通信不良や誤入力を起こした際にも、安全に対処できる運用体制の確立を目指します。

### 現場運用フロー概要（クイックリファレンス）

| 場面 | 職員の操作 | システム挙動 | 注意点 |
| -------------- | ------------------ | ----------- | --------------- |
| トイレ記録を登録 | 利用者・種別・量・メモを入力して保存 | 記録を保存し一覧を更新 | 保存中は二重押下しない |
| 記録を訂正 | 訂正可能項目のみ変更 | 訂正後に再取得 | 利用者・日時・記録者は変更不可 |
| 保存失敗 | エラー表示を確認 | 保存は確定していない | 再送前に重複確認 |
| Kioskから退出 | 退出FAB等を使う | 通常画面へ戻る | 共有端末の放置に注意 |
| 保存制限時 (Write Disabled) | 入力・保存を控える | 警告バナー表示 | 管理者確認が必要 |

## 2. Kioskモードの位置づけ
キオスクモードは、忙しい日中支援の合間に、職員がその場で素早く支援内容を記録するための「共有端末専用の全画面インターフェース」です。
通常のPC画面（管理・計画書作成・分析用）と比べて、以下の工夫が施されています。
* **情報のシンプル化**: メニューやヘッダーを極力排除し、画面いっぱいに大きなボタンと利用者の名前を表示します。
* **操作ミスの防止**: 利用者の誤タップによる画面の閉じや、意図しない設定変更を防ぐための画面ロック機能（長押しによる終了）を備えています。

---

## 3. 対象ユーザーと利用場面
* **対象者**: 施設内の直接支援に携わる介護員、生活支援員、指導員。
* **利用場面**:
  1. **朝の通所時**: 利用者が到着した際に出欠（通所）を登録します。
  2. **日中活動中**: 各利用者に設定された個別の「支援手順」を実施した際、その進捗を記録します。
  3. **トイレ介助時**: 排泄の状況（種別、量、時間）をその都度リアルタイムで入力します。
  4. **夕方の退所時**: 利用者が帰宅する際に退所登録を行います。

---

## 4. 画面導線
キオスクモードを起動すると、以下のホーム画面が表示され、4つの主操作へワンタップで進むことができます。

```
【キオスクホーム画面】
┌──────────────────────────────────────────────┐
│  キオスクモード                              │
│                                              │
│  [ 支援手順を実施する ] ── 最優先アクション  │
│                                              │
│  [ 通所する ]             [ 退所する ]       │
│                                              │
│  [ 今日の予定を見る ]                        │
└──────────────────────────────────────────────┘
```
※画面の右下には、目立たない形で「退出ボタン（長押しメニュー）」が常駐しています。

## 5. トイレ記録の操作手順

トイレ記録画面（本日のトイレ確認）は、職員の記録忘れ（取りこぼし）を防ぐ工夫が組み込まれています。

### 1. 未記録者の案内パネル（ガイダンス機能）
* 画面最上部に「未記録者が〇名います。次は〇〇さんを記録すると取りこぼしを減らせます」という案内パネルが表示されます。この案内に従って記録を行うことで、全員の記録漏れを防ぐことができます。
* 全員の入力が完了すると、自動的に「本日の対象者は全員記録済みです」という緑色の完了メッセージに切り替わります。

### 2. 新規の記録入力手順
1. 該当する利用者の行にある **「記録する」** ボタン（または追加で記録する場合は **「追加記録」** ボタン）を押します。
2. 入力画面（ダイアログ）が開いたら、以下の項目をセットします。
   - **時間**: 自動で現在時刻が入ります（必要に応じて前後できます）。
   - **種類**: 「排尿」「排便」「両方」「その他」から選びます。
   - **量**: 「少量」「普通」「多量」「不明」から選びます。
   - **メモ**: 固形物の状態や特記すべき様子があれば自由に入力します。
3. **「保存」** ボタンを押すと記録が追加され、一覧に「済（緑色）」マークがつきます。

## 6. 食事・その他記録の操作手順
日中活動中の水分量や食事、その他個別の手順記録についても、キオスクホームの「支援手順を実施する」から利用者を選択し、あらかじめ登録された手順ステップに従って実行チェックを入れ、完了ボタンを押すだけで記録が保存されます。
画面のステップはすべてタブレットで押しやすいよう設計されており、複雑な文章入力をすることなく、完了・スキップの判定を記録できます。

---

## 7. 訂正操作の範囲と禁止事項
入力した記録を修正する際は、一覧表の「訂正」ボタンから行います。ただし、監査対応と記録の信頼性を確保するため、変更できる範囲に厳格な境界が設けられています。

### 1. 訂正できる項目 (変更可能)
* **種類** (排尿・排便などの切り替え)
* **量** (少量・普通などの変更)
* **メモ** (記述内容の修正や追記)

### 2. 訂正できない項目 (完全ロック・変更不可)
* **対象の利用者** (誤った利用者の行で記録した場合は、削除して再入力が必要です)
* **記録日・記録日時** (記録の発生時刻は改ざん防止のため変更できません)
* **記録者名** (キオスク端末からの入力であるため、入力者名は自動的に固定されます)

## 8. 保存失敗・通信不良・Write Disabled時の対応

ネットワークの瞬断や、システムの保守保護状態（保存機能制限）が発生した場合の対処手順です。

### 1. 二重送信の自動防止機能
* 職員が「保存」をタップすると、ボタンが自動的に「保存中...」に切り替わり、保存処理が完了するまで追加のタップは受け付けなくなります（二重送信防止ガード）。
* 連打による重複データの作成を防ぐ設計になっていますので、ボタンが元に戻るか画面が切り替わるまでそのままでお待ちください。

### 2. 「通信エラー」で保存が失敗した場合の対応
* 通信エラーやサーバー接続不良のアラートが表示された場合、**記録は確定していません**。
* **対処手順**:
  1. 直ちに再度「保存」を押さず、一度入力画面を閉じます。
  2. 一覧画面の最下部にある「本日の全記録（個人別）」の履歴欄を確認します。
  3. 既に記録が一覧に反映されている（背後で送信が成功していた）場合は、再入力の必要はありません。反映されていない場合のみ、再度「記録する」から入力を行ってください。

### 3. 「書き込み保護（保存停止）」バナーが表示されている場合
* システムの保守中や環境設定の制限により、画面上部に「保存機能が無効になっています（書き込み制限）」というオレンジ色の警告バナーが表示されることがあります。
* このバナーが表示されている間は、画面上での入力は可能に見えても、実際の保存はすべてブロックされます。
* **対処手順**:
  - タブレットからの入力作業を一時中断し、施設のシステム管理者に「書き込み制限バナーが表示されている」旨を報告して指示を仰いでください。

## 9. 共有端末運用上の注意点（いたずら・誤操作防止）

共有タブレットをホールなどに設置する際、利用者が画面を触ってキオスクモードを終了させてしまったり、ブラウザの他のページを開いてしまうリスクがあります。これに対応するため、独自のロック解除方式が採用されています。

### 1. キオスク退出の制限 (長押し FAB)
* 通常の画面とは異なり、メニューバーや戻るボタンはありません。
* キオスクモードを終了して通常の管理画面（PC版レイアウト）に戻るには、画面右下の終了アイコンを **「1.5秒以上長押し」** する必要があります。
* 長押ししている間、ボタンの周りに円形のゲージ（プログレスバー）が表示され、完全に円が閉じるまで押し続けるとメニューが開きます。
* メニューが開いた際、**「通常モードに戻る」** を選択することで退出できます。
* ※ゲージの途中で指を離すとキャンセルされ、メニューは開きません。

### 2. 画面の再読み込み
* 画面の表示が崩れたり、動きが止まった（フリーズした）ように見える場合は、上記長押しメニューから **「再読み込み」** を選択することで、画面全体が初期リフレッシュされます。

## 10. 現場導入前チェックリスト
導入前に、以下の項目が準備できているか確認してください。

- [ ] **端末設置位置**: 利用者が歩行時にぶつからず、かつ職員が介助後すぐにアプローチできる安全な場所にタブレットを固定しているか。
- [ ] **通信の確認**: 設置場所でWi-Fi接続が安定しており、SharePoint等のデータ接続が遮断されていないか。
- [ ] **退出操作の訓練**: 導入に関わる全職員が「1.5秒長押し」によるキオスクモード退出手順と、二重送信防止時の確認方法（履歴欄のチェック）を理解しているか。
- [ ] **書き込み保護時の連絡網**: 画面に制限警告（Write Disabledバナー等）が出た場合の管理者への連絡フローが確立しているか。

## 11. 今後の改善ロードマップ (Next Small PRs)

1. **`kiosk: add-custom-passcode-for-exit` (中規模)**
   * **目的**: 現在の1.5秒長押しによる退出に加え、職員専用の暗証番号（パスコード）の入力を必須にすることで、より厳密に一般利用者による誤操作や無断退出を防ぐ。
2. **`ui: add-save-retry-cache` (大規模)**
   * **目的**: 通信エラーが発生した際、入力されたデータを一時的にブラウザメモリへ退避させ、通信復旧時に自動で再送処理を行う「オフライン一時保存機能」を追加する。
3. **`kiosk: auto-reset-to-home` (小規模)**
   * **目的**: 個別の入力画面を開いたまま職員が離席し、一定時間（例: 3分間）操作がない場合に、自動でキオスクホーム画面へ戻るオートリセット処理を組み込み、画面の放置を防ぐ。
