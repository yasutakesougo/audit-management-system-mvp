# Index Advisor 運用ルール定義

Index Advisor の診断および修復結果に基づき、Nightly Patrol が生成する重要度別の運用アクションを以下に定義します。

## 1. 運用重要度と対応方針

| 重要度 | ラベル | Nightly 表示 | 対応方針 |
| :--- | :--- | :--- | :--- |
| **critical** | 🔴 致命的 | 即日対応 | **必須インデックスの不足。** スループット低下や 5000 件制限超過のリスクがあるため、当日中に Index Advisor を実行して修復してください。 |
| **action_required** | 🟠 要対応 | 権限/設定確認 | **修復アクションの失敗。** 自動修復がエラーになった場合に表示されます。SharePoint の権限設定や、リストの状態を直接確認してください。 |
| **watch** | 🟡 監視 | 計画対応 | **候補インデックスの不足。** 将来のパフォーマンス向上のための提案です。次回のメンテナンス時などに Index Advisor での修復を検討してください。 |
| **silent** | 🟢 正常 | 吸収済み | **修復成功または正常。** 背景で行われた自動修復の履歴です。システムが自律的に解決しており、管理者による対応は不要です。 |

## 2. Nightly Summary における判別

管理者は朝のレポートを確認する際、以下の「NextAction」に従ってください。

*   **「至急 Index Advisor で修復を実行してください」**
    *   対象リストのデータ量が増える前に対応が必要です。
*   **「インデックス修復アクションに失敗しました」**
    *   システムによる自動修復が阻害されています。管理者アカウントでの手動確認が必要です。
*   **「Index Advisor を確認してください」**
    *   システムの健康状態は維持されていますが、中長期的な最適化が可能です。

## 3. 監査トレース
すべての修復（Remediation）アクションは `DriftEventsLog` に記録され、過去の修復履歴を遡ることが可能です。
