# spClient.ts の God File 分割

- **対象ファイル**: `src/lib/spClient.ts`（1,630行）
- **カテゴリ**: リファクタリング
- **現状の課題**:
  `spClient.ts` は 1,630 行に膨れ上がっており、SharePoint接続、リトライロジック、キャッシュ、バッチ処理、各種ユーティリティなどが単一ファイルに集約されています。
  - 認知負荷が非常に高く、新規メンバーが修正箇所を特定しにくい
  - 変更時の影響範囲が予測困難（リトライ修正がキャッシュに影響するなど）
  - テストのセットアップが複雑で、単体テストが書きにくい
  - ファイルサイズガード（600行上限）を大幅に超過
- **解決策の提案**:
  以下のモジュールに分割する:
  ```
  src/lib/sp/
  ├── spClient.ts          # コアの SP 接続 & useSP フック (≤200行)
  ├── spRetry.ts           # リトライ / 429 バックオフロジック
  ├── spBatch.ts           # バッチリクエスト処理
  ├── spCache.ts           # キャッシュ戦略
  ├── spListOps.ts         # リスト CRUD ヘルパー
  └── index.ts             # re-export barrel
  ```
  各モジュールを単一責任に保ち、既存の `import { useSP } from '@/lib/spClient'` は barrel 経由で互換維持する。
- **見積もり影響度**: High
