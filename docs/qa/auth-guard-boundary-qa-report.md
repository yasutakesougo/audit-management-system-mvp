# Auth Guard Boundary QA Report

## 検証目的

`ProtectedRoute` と `RequireAudience` のバイパス判定ロジック不一致に起因していた、
**「ログインを確認中…」画面でのハングアップ現象** が、実機で解消されていることを確認する。

## 合否基準

**OK**

* `ログインを確認中…` で停止しない
* 認証が必要な場合、適切にログイン導線へ進む
* bypass 条件では意図どおり描画される
* 復帰後に白画面・無反応・再読込ループが発生しない

**NG**

* 5秒以上 `ログインを確認中…` が継続する
* `reason` が短時間に不自然に揺れる
* 復帰後に認証状態が破綻する
* 認証導線と bypass の挙動が環境ごとに矛盾する
