---

## 概要

本ロードマップは、[設計原則 10 箇条](principles.md) を実装に接続するために、
機能を **フェーズ単位** で優先順位付けしたものである。

基本思想は **原則 7: まず日次価値、次に月次価値** に従い、
現場が毎日使う仕組みから順に価値を積み上げる。

---

## フェーズ構成

```
Phase 1: Daily Value（日次価値）          ← 現場定着の土台
Phase 2: Monthly Intelligence（月次知性）  ← PDCA サイクル完成
Phase 3: Unified Proposal Layer（統合提案） ← 判断補助 OS 始動
Phase 4: Organizational Knowledge（組織知） ← 知識基盤の構築
Phase 5: Platform Evolution（OS 進化）     ← スケーラビリティ
```

---

## 成功指標

### Phase 1 完了基準

- [ ] 現場職員の 80% 以上が毎日 Today 画面を開く
- [ ] 申し送りが紙運用から完全に移行された
- [ ] 記録入力の平均時間が手書きと同等以下

### Phase 2 完了基準

- [ ] モニタリング下書きの自動生成率 70% 以上
- [ ] Provenance 追跡率 100%（全取り込みフィールド）
- [ ] 90 日サイクルの見落とし件数がゼロ

### Phase 3 完了基準

- [ ] 全提案ソースが統一 UI で表示される
- [ ] 採用/却下に根拠と理由が記録される
- [ ] 提案の確認率 90% 以上（expired が 10% 未満）

### Phase 4 完了基準

- [ ] 採用パターンレポートが自動生成される
- [ ] 新人教育に採用事例集が活用される
- [ ] 監査時に根拠付き運用履歴を即時提示可能
