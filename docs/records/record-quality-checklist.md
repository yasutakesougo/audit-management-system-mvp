# 支援記録 品質チェックリスト

## 目的

このチェックリストは、日々の支援記録を後から読み返しやすくし、モニタリング会議や個別支援計画の準備に使いやすくするための確認観点である。

チェック結果は支援記録の補助情報であり、人間レビューのために使う。AIだけで記録を確定したり、支援方針を決めたりしてはいけない。

## 基本姿勢

- 観察できた事実を中心に書く。
- 本人の意思表示、職員の支援、支援後の反応を分けて書く。
- あいまいな言葉は、具体的な場面、行動、声かけ、反応に置き換える。
- 記録にない理由や背景は断定しない。
- 敬意のある中立的な表現を使う。

## チェック項目

- 場面や状況が分かるか。
- 時刻、時間帯、タイミングが分かるか。
- 利用者の観察可能な行動が書かれているか。
- 職員が行った支援が書かれているか。
- 支援後の利用者の反応が書かれているか。
- 関係する環境要因がある場合、それが書かれているか。
- 事実と解釈が分かれているか。
- 「不安定」「いつも通り」「拒否」「パニック」などの表現が、より具体的に書けるか。
- 次回確認や申し送りが必要な場合、その確認観点があるか。
- 原本の記録をAI出力で置き換えていないか。

## 表現を具体化する観点

| あいまいな表現 | 確認したい観点 |
| --- | --- |
| 不安定だった | どの場面で、どのような表情、発言、動きがあったか |
| いつも通り | 何が普段と同じだったか。活動、食事、会話、休憩など |
| 拒否した | 何に対して、どのように意思表示したか |
| パニックになった | どの場面で、どのような行動や声があり、職員が何をしたか |
| 落ち着いた | どの支援後に、どのような様子になったか |
| 問題なし | 確認した項目と、観察できた状態は何か |

## 低情報の記録例と改善例

### 例1: 活動参加

低情報:

> 午前の活動は不安定で参加できなかった。

改善例:

> 10:20頃、作業室で紙折りの活動を案内した。本人は席に座った後、机から離れて入口付近に移動した。職員が「見学からでも大丈夫です」と声をかけると、5分ほど入口付近で活動を見た。その後、本人から「少しやる」と発言があり、11:00まで職員と一緒に3枚取り組んだ。

確認観点:

- 活動前に予定変更や環境変化があったか。
- 次回も見学から始める選択肢が有効か、人間レビューで確認する。

### 例2: 食事

低情報:

> 昼食をあまり食べなかった。

改善例:

> 12:10から昼食。主食は半分程度、副菜は数口、水分はコップ半分程度。本人は「今日はもういい」と話した。職員が休憩を提案すると、本人は食堂横の椅子で10分休憩した。13:00の活動前には表情が和らぎ、職員の声かけにうなずいて参加した。

確認観点:

- 体調、眠気、食事内容の好みなど、次回確認したい点があるか。
- 家族共有が必要かは人間レビューで判断する。

### 例3: トイレ誘導

低情報:

> トイレ拒否あり。

改善例:

> 14:30、外出前に職員がトイレに行くか確認した。本人は首を横に振り、「行かない」と発言した。職員が「出発前にもう一度聞きます」と伝えると、本人はうなずいた。14:45に再度確認すると、本人から「行く」と発言があり、職員見守りでトイレを使用した。

確認観点:

- 声かけのタイミングや伝え方が本人に合っていたか。
- 次回も外出前の確認方法を統一するか、人間レビューで確認する。

## AIが出すべき補助情報

- 仕分け候補
- 不足情報
- 確認観点
- 原本のどの表現を具体化できそうか
- 人間レビューで確認すべき点

## AIが出してはいけない補助情報

- 利用者の診断名や特性の断定
- 行動の価値判断
- 支援方針の確定
- 職員や利用者の評価スコア
- 原本の上書き案としての確定文
