# 📋 支援記録（19行記録）SharePoint 永続化 移行検証チェックリスト

本ドキュメントは、Power Apps から React への支援記録移行を安全に完結させるための検証項目をまとめたものです。

---

## 🎯 1. SharePoint リスト構成の最終確認

保存先リストが正しく構成されているか、以下の項目を確認してください。

- [ ] **子リスト設定**: `SupportRecord_DailyRows` (or VITE_SP_LIST_DAILY_ROWS)
  - [ ] `RowKey` 列（単一行テキスト）が存在し、インデックス化されていること
  - [ ] `ParentIDLookup` 列（数値 or ルックアップ）が存在し、インデックス化されていること
  - [ ] `RowNo` 列（数値 or テキスト）が存在すること
- [ ] **親リスト設定**: `SupportRecord_Daily` (or VITE_SP_LIST_DAILY)
  - [ ] `Title` 列に `${date}-${userId}` 形式のキーが保存されること
  - [ ] `RecordDate` 列（日付のみ）が存在すること
- [ ] **権限**: 入力ユーザーが両方のリストに対して「投稿（編集）」権限を持っていること

---

## 🔍 2. データ整合性の検証（単体テスト）

新しいリポジトリ経由での保存・取得が、期待通り親子関係を維持しているか確認します。

- [ ] **親レコードの自動作成**:
  - `SupportRecord_Daily` に対象日のレコードがない状態で1行保存し、親が作成されるか
- [ ] **子レコードのUpsert (更新)**:
  - 同じ時間スロットを2回保存（完了 → スキップなど）したとき、`DailyRows` に新規追加されず、既存行が更新されるか（`RowKey` 検索が効いているか）
- [ ] **親子紐付け**:
  - `DailyRows` の `ParentIDLookup` が、`SupportRecord_Daily` の正しいレコード ID を指しているか
- [ ] **5000件閾値回避**:
  - `getRecords` / `getRecord` が `Title` ではなく `RowKey` (インデックス付き) でフィルタリングされているか
- [ ] **親サマリー更新**:
  - 子レコード保存後、`SupportRecord_Daily` の進捗件数・完了率が必要に応じて更新されるか


---

## 🛠️ 3. 実機操作・UXフローの検証

React 画面上での操作性が壊れていないか確認します。

- [ ] **ステータス反映**:
  - `ExecutionToggle` をクリックした後、アイコンが正しく切り替わり、リロード後も維持されるか
- [ ] **非同期ロード表示**:
  - ページを開いた際、SharePoint からのデータ取得中に適切なロード状態（スケルトンなど）が表示されるか
- [ ] **オートアドバンス**:
  - 1行入力後に自動で次のスロットへ移動する挙動が、非同期保存を待ってから行われているか
- [ ] **日報コピー**:
  - 「日報をコピー」ボタンを押した際、SharePoint 上の最新の記録（全19行分）が正しくレポートに含まれるか
- [ ] **利用者別テンプレート差異**:
  - 19行固定でない利用者テンプレートでも、行番号・保存・再表示が破綻しないか


---

## 🧪 4. 並行運用・切り替えプロトコル

Power Apps とのデータの整合性を維持しながら切り替える手順です。

- [ ] **列名の完全一致**:
  - `constants.ts` で定義した `DAILY_RECORD_FIELDS` / `ExecutionRecord_FIELDS` が、既存の Power Apps が使用している列名と一致しているか
- [ ] **フェーズド・ロールアウト**:
  - [ ] Step 1: 特定のテスト用ユーザーのみ React で入力し、SharePoint 側の実データをエンジニアが目視確認
  - [ ] Step 2: 過去の記録（履歴）が React 側の親子構造リポジトリでも正しく表示されるか確認
  - [ ] Step 3: 同じ利用者・同じ日付について、React と Power Apps の両方から同時更新しない運用ルールが定義されているか
  - [ ] Step 4: Power Apps 側の「保存」ボタンを無効化し、React へ完全移行


---

## 🚨 トラブルシューティング（異常系の確認）

- [ ] **ネットワークエラー**:
  - 保存中にオフラインになった場合、画面上に「保存失敗」等のスナックバーが出るか
- [ ] **二重作成競合**:
  - 複数のタブやデバイスで同時に同じ日の親レコードを作成しようとしたとき、システムがクラッシュせず、一方の ID を再利用するか
