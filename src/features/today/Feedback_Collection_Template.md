# フィードバック収集・「違和感ログ」テンプレート

現場テストで出た「声」を、エンジニアが次の「削り」に繋げるためのフォーマットです。

## 📝 入力フォーマット

### 【違和感タイトル】
（例：保存後に一瞬だけ管理画面が見えてしまう、など）

### 1. 発生タイミング
- [ ] ログイン直後
- [ ] 記録入力中
- [ ] 保存ボタン押下直後
- [ ] Today Hub 閲覧中

### 2. 現場の声（そのまま引用）
「〜〜だと思った」「〜〜が怖かった」などの感情を優先して記録してください。

### 3. エンジニア向け分析
- **原因**: (例) ローディング状態の制御不足
- **重要度**: [ ] 高（現場が止まる） / [ ] 中（不安が残る） / [ ] 低（見た目の微調整）
- **対策案**: (例) スケルトンスクリーンの延長、または特定の role での非表示化

---

## 🚀 次に「削る」候補リスト（叩き台）
テスト中に以下の反応があれば、チェックを付けてください。
- [ ] 右上の「通知」アイコンは見ていない（→ 現場 role では削除検討）
- [ ] サイドバーの「設定」を間違えて押している（→ 現場 role では非表示）
- [ ] 下部の「システム稼働状況」に驚いている（→ 現場 role では削除）
